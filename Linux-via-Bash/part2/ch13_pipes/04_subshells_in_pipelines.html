Подпроцессы в конвейере

Подпроцессы

Каждая команда в многокомандном конвейере выполняется в своём подпроцессе (subshell):

# Каждая команда в отдельном процессе
command1 | command2 | command3

Подпроцесс — это отдельный процесс, копия родительского shell.

Проверка PID

# Разные PID для каждой команды
bash -c 'echo "Parent: $$"' | \
bash -c 'echo "First: $$"' | \
bash -c 'echo "Second: $$"'

Последствия подпроцессов

Переменные не сохраняются

# Переменная теряется
echo "data" | read x
echo "$x"  # Пусто!

# Потому что read выполнен в подпроцессе

Решение: lastpipe

# Включить lastpipe
shopt -s lastpipe
set +m  # Отключить job control

# Последняя команда выполняется в текущем shell
echo "data" | read x
echo "$x"  # "data"

lastpipe

Опция lastpipe позволяет последней команде выполняться в процессе shell:

# Требования для lastpipe:
# 1. shopt -s lastpipe
# 2. job control отключен (set +m)

shopt -s lastpipe
set +m

# Теперь работает
echo "value" | read var
echo "$var"  # "value"

Примеры

Проблема с подпроцессами

# Счётчик не работает
count=0
cat file.txt | while read line; do
    ((count++))
done
echo $count  # 0! (изменения в подпроцессе)

Решение

# Использовать перенаправление вместо конвейера
count=0
while read line; do
    ((count++))
done < file.txt
echo $count  # Правильное значение

Или с lastpipe

shopt -s lastpipe
set +m

count=0
cat file.txt | while read line; do
    ((count++))
done
echo $count  # Правильное значение

Асинхронные конвейеры

Если конвейер запущен в фоне (&), оболочка не ждёт завершения:

# Асинхронный конвейер
command1 | command2 &

# Код выхода = 0
echo $?  # 0

Практические примеры

Подсчёт строк

# Неправильно (подпроцесс)
count=0
ls | while read file; do
    ((count++))
done

# Правильно (перенаправление)
count=0
while read file; do
    ((count++))
done < <(ls)

Сбор данных

# С lastpipe
shopt -s lastpipe
set +m

result=""
echo "data" | result=$(cat)
echo "$result"  # "data"

Параллельная обработка

# Конвейер для параллелизма
cat file.txt | \
    grep "pattern" | \
    sort | \
    uniq -c | \
    sort -rn > output.txt
