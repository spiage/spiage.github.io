От основ Linux к мастерству Bash

Специальные встроенные команды

Специальные builtins — это встроенные команды с особым поведением при обработке ошибок и переменных.

Что такое специальные builtins

Специальные встроенные команды обрабатываются оболочкой особым образом:

  1. Ошибки — ошибка в специальной команде завершает неинтерактивную оболочку
  2. Переменные — специальные builtins выполняются до раскрытия переменных
  3. Перенаправления — перенаправления обрабатываются до выполнения команды

Команды

: (двоеточие)

Пустая команда — не делает ничего:

# Бесконечный цикл
while :; do
    echo "Working..."
    sleep 1
done

# Заполнитель (когда синтаксис требует команду)
if condition; then
    : # TODO: реализовать позже
fi

# С перенаправлением — создать пустой файл
: > empty_file

. (точка, source)

Выполняет файл в текущей оболочке:

# Загрузить переменные окружения
. ~/.bashrc
source ~/.bashrc  # Синоним

# С аргументами
source script.sh arg1 arg2

# С поиском в PATH
. myscript  # Ищет в PATH

break

Выходит из цикла:

# Выйти из цикла
while true; do
    read input
    [[ $input == "quit" ]] && break
    echo "You entered: $input"
done

# Выйти из N уровней вложенных циклов
for i in {1..3}; do
    for j in {1..3}; do
        [[ $j -eq 2 ]] && break 2  # Выход из обоих циклов
        echo "i=$i, j=$j"
    done
done

continue

Переходит к следующей итерации:

# Пропустить чётные числа
for i in {1..10}; do
    (( i % 2 == 0 )) && continue
    echo "$i is odd"
done

eval

Объединяет и выполняет аргументы:

# Динамическое выполнение
cmd="echo"
arg="Hello"
eval $cmd $arg  # Выполняет: echo Hello

# Создание переменных
varname="count"
eval $varname=5
echo $count  # 5

# Опасно с пользовательским вводом!
user_input="; rm -rf /"
eval echo $user_input  # Выполнит rm!

exec

Заменяет процесс или перенаправляет FD:

# Заменить оболочку другой программой
exec /bin/bash  # Текущий процесс заменяется на bash

# Перенаправление для всей оболочки
exec > output.log  # Весь вывод идёт в файл
exec 2>&1          # stderr в stdout

# Открыть файловый дескриптор
exec 3< input.txt  # FD 3 для чтения
exec 4> output.txt # FD 4 для записи

exit

Выходит из оболочки:

# Выйти с кодом 0
exit 0

# Выйти с кодом ошибки
exit 1

# Использовать статус последней команды
exit

export

Экспортирует переменные в окружение:

# Экспортировать переменную
export PATH=/usr/bin:$PATH
export MYVAR="value"

# Экспортировать функцию
export -f my_function

# Показать экспортированные
export -p

readonly

Делает переменную только для чтения:

# Сделать переменной только для чтения
readonly PI=3.14159
PI=3  # Ошибка!

# Показать read-only переменные
readonly -p

return

Возвращается из функции:

my_function() {
    [[ -z $1 ]] && return 1
    echo "Argument: $1"
    return 0
}

set

Устанавливает опции оболочки:

# Включить опции
set -e  # Выход при ошибке
set -u  # Ошибка при неопределённой переменной
set -x  # Отладка

# Выключить опции
set +e
set +u
set +x

# Показать опции
set -o

shift

Сдвигает позиционные параметры:

# Обработка аргументов
while [[ $# -gt 0 ]]; do
    case $1 in
        -v) verbose=1; shift ;;
        -f) file=$2; shift 2 ;;
        *) echo "Unknown: $1"; shift ;;
    esac
done

trap

Устанавливает обработчики сигналов:

# Обработка прерывания
trap 'echo "Interrupted!"; exit 1' INT

# Очистка при выходе
trap 'rm -f /tmp/tempfile' EXIT

# Показать trap
trap -p

unset

Удаляет переменные или функции:

# Удалить переменную
unset MYVAR

# Удалить функцию
unset -f my_function

# Удалить экспортированную
unset -n VARNAME

Особенности специальных builtins

Ошибки завершают оболочку

# В неинтерактивной оболочке:
#!/bin/bash
exit wrong_arg  # Ошибка — оболочка завершается
echo "Не выполнится"

Переменные раскрываются после

# Специальный builtin выполняется до раскрытия
export VAR=value
echo $VAR  # Работает

# Но:
set -- a b c
echo $1  # a (раскрылось после set)
