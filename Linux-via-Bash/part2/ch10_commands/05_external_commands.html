От основ Linux к мастерству Bash

Внешние команды

Внешние команды — это отдельные исполняемые файлы, которые Bash находит и запускает.

Как Bash находит внешние команды

Порядок поиска

  1. Псевдонимы (alias)
  2. Ключевые слова (if, for, while)
  3. Функции
  4. Встроенные команды (builtins)
  5. Внешние команды (в PATH)

Переменная PATH

# Показать PATH
echo $PATH
# /usr/local/bin:/usr/bin:/bin:/usr/games

# Добавить в PATH
export PATH=$PATH:/opt/mybin

# Временное изменение
PATH=/usr/bin:/bin command

Поиск команд

type

# Показать тип и путь
type ls
# ls is hashed (/bin/ls)

type -a ls
# ls is /bin/ls
# ls is /usr/bin/ls

which

# Найти путь
which ls
# /bin/ls

# Несколько команд
which ls grep find

whereis

# Найти бинарник и man
whereis ls
# ls: /bin/ls /usr/share/man/man1/ls.1.gz

# Только бинарник
whereis -b ls

# Только man
whereis -m ls

hash

Bash кэширует пути к командам:

# Показать кэш
hash

# Очистить кэш
hash -r

# Удалить конкретную команду
hash -d ls

# Статистика
hash -v
# hits    command
#    5    /bin/ls

Выполнение команд

Прямой вызов

# Абсолютный путь
/bin/ls -la

# Относительный путь
./myscript.sh

# Из PATH
ls -la

С подстановкой

# Подстановка команд
echo "Files: $(ls | wc -l)"

# Обратные кавычки (устарело)
echo "Files: `ls | wc -l`"

Примеры использования

Конвейеры

# Найти и отфильтровать
ps aux | grep python | grep -v grep

# Посчитать строки
find . -name "*.txt" | wc -l

# Сортировать и уникальные
cat file.txt | sort | uniq

xargs

# Удалить найденные файлы
find . -name "*.tmp" | xargs rm

# С ограничением аргументов
find . -name "*.log" | xargs -n 10 rm

# С разделителем NUL
find . -name "*.txt" -print0 | xargs -0 rm

tee

# Вывод на экран и в файл
ls -la | tee output.txt

# Добавление в файл
ls -la | tee -a output.txt

Оптимизация

Избегание лишних вызовов

# Плохо: много внешних вызовов
for file in *.txt; do
    wc -l $file
done

# Лучше: один вызов
wc -l *.txt

Встроенные vs внешние

# Внешняя команда (медленнее)
/bin/echo "Hello"

# Встроенная (быстрее)
echo "Hello"

Кэш hash

# Первый вызов — поиск в PATH
ls

# Последующие — из кэша
ls

# Очистить кэш при изменении PATH
hash -r

Отладка

strace

Трассировка системных вызовов:

# Трассировать команду
strace ls

# Только open вызовы
strace -e open ls

ltrace

Трассировка библиотечных вызовов:

ltrace ls

time

Измерение времени:

# Простое время
time ls

# Подробное
/usr/bin/time -v ls
