Просмотр содержимого файлов

В этом разделе рассматриваются команды для вывода содержимого файлов: от простого отображения всего файла до постраничного просмотра и вывода частей файла.

cat — конкатенация и вывод файлов

cat (concatenate) читает файлы и выводит их содержимое на стандартный вывод.

Синопсис

cat [ОПЦИЯ]... [ФАЙЛ]...

Описание

Читает ФАЙЛы последовательно и выводит их содержимое. Если ФАЙЛ не указан или равен -, читает стандартный ввод.

Опции

  • -A, --show-all — показать все символы (эквивалентно -vET)
  • -b, --number-nonblank — пронумеровать непустые строки
  • -e — показать символы конца строки и непечатаемые (эквивалентно -vE)
  • -E, --show-ends — показывать $ в конце каждой строки
  • -n, --number — пронумеровать все строки
  • -s, --squeeze-blank — сжимать несколько пустых строк в одну
  • -t — показать табуляции и непечатаемые (эквивалентно -vT)
  • -T, --show-tabs — показывать табуляции как ^I
  • -v, --show-nonprinting — показывать непечатаемые символы как ^M и M-
  • --help — вывести справку
  • --version — вывести версию

Примеры

# Вывести содержимое файла
cat file.txt

# Вывести несколько файлов подряд
cat file1.txt file2.txt

# Пронумеровать все строки
cat -n file.txt

# Пронумеровать только непустые строки
cat -b file.txt

# Показать непечатаемые символы
cat -A file.txt

# Сжать пустые строки
cat -s file.txt

# Читать из stdin и вывести
cat - file.txt

# Создать новый файл
cat > newfile.txt

# Добавить в конец файла
cat >> existing.txt

Особенности

Чтение из stdin

Если файл не указан или имя файла -, cat читает из стандартного ввода:

# Конвейер
echo "hello" | cat

# Несколько входов
cat file1.txt - file2.txt

Бинарные файлы

cat может выводить бинарные файлы, но это может повредить терминал:

# ОПАСНО: может повредить терминал
cat binary_file

# Безопасно: использовать less или hexdump
less binary_file
hexdump -C binary_file

Конкатенация файлов

cat часто используется для объединения файлов:

# Объединить файлы
cat part1.txt part2.txt > combined.txt

# Разделить и объединить
split largefile.txt part_
cat part_* > largefile.txt

Коды возврата

  • 0 — успешно
  • 1 — ошибка при чтении файла

head — вывод начала файла

head выводит первые строки файлов.

Синопсис

head [ОПЦИЯ]... [ФАЙЛ]...

Описание

Выводит первые строки каждого ФАЙЛа. По умолчанию выводит первые 10 строк.

Опции

  • -c, --bytes=[-]N — вывести первые N байт
  • -n, --lines=[-]N — вывести первые N строк (по умолчанию 10)
  • -q, --quiet, --silent — не выводить заголовки с именами файлов
  • -v, --verbose — всегда выводить заголовки с именами файлов
  • -z, --zero-terminated — разделитель строк — NUL, а не newline
  • --help — вывести справку
  • --version — вывести версию

Примеры

# Вывести первые 10 строк файла (по умолчанию)
head file.txt

# Вывести первые 20 строк
head -n 20 file.txt

# Вывести первые 100 байт
head -c 100 file.txt

# Вывести несколько файлов
head file1.txt file2.txt

# Вывести без заголовков
head -q *.txt

# Вывести всё кроме последних 5 строк
head -n -5 file.txt

# Вывести первые 5 строк из каждого файла
head -n 5 *.log

Особенности

Отрицательное значение

Отрицательное N выводит всё кроме последних N строк/байт:

# Вывести всё кроме последних 5 строк
head -n -5 file.txt

# Вывести всё кроме последних 100 байт
head -c -100 file.txt

Несколько файлов

При выводе нескольких файлов head добавляет заголовок:

head file1.txt file2.txt
# ==> file1.txt <==
# первые 10 строк file1.txt
#
# ==> file2.txt <==
# первые 10 строк file2.txt

Конвейеры

head часто используется в конвейерах для ограничения вывода:

# Первые 10 строк вывода
ls -la | head

# Первые 5 совпадений
grep "pattern" file.txt | head -n 5

# Первые 100 байт
cat file.txt | head -c 100

Коды возврата

  • 0 — успешно
  • 1 — ошибка при чтении

tail — вывод конца файла

tail выводит последние строки файлов.

Синопсис

tail [ОПЦИЯ]... [ФАЙЛ]...

Описание

Выводит последние строки каждого ФАЙЛа. По умолчанию выводит последние 10 строк.

Опции

Основные опции

  • -c, --bytes=[-]N — вывести последние N байт
  • -n, --lines=[-]N — вывести последние N строк (по умолчанию 10)
  • -q, --quiet, --silent — не выводить заголовки с именами файлов
  • -v, --verbose — всегда выводить заголовки с именами файлов
  • -z, --zero-terminated — разделитель строк — NUL, а не newline

Режим отслеживания

  • -f, --follow[={name|descriptor}] — отслеживать добавление данных в файл
  • -F — эквивалентно --follow=name --retry
  • --max-unchanged-stats=N — количество изменений перед проверкой файла (по умолчанию 5)
  • --pid=PID — завершиться после завершения процесса PID
  • --retry — продолжать попытки открыть файл
  • --sleep-interval=N — интервал проверки в секундах (по умолчанию 1)

Примеры

# Вывести последние 10 строк файла (по умолчанию)
tail file.txt

# Вывести последние 20 строк
tail -n 20 file.txt

# Вывести последние 100 байт
tail -c 100 file.txt

# Вывести несколько файлов
tail file1.txt file2.txt

# Отслеживать изменения файла (лог-файлы)
tail -f /var/log/syslog

# Отслеживать с повторными попытками открытия
tail -F /var/log/app.log

# Вывести всё кроме первых 5 строк
tail -n +5 file.txt

# Отслеживать несколько файлов
tail -f /var/log/syslog /var/log/auth.log

# Отслеживать с интервалом 5 секунд
tail -f -s 5 file.log

# Завершиться после завершения процесса
tail -f --pid=$$ file.log

Особенности

Положительное значение с +

Положительное N с + выводит всё кроме первых N строк/байт:

# Вывести всё кроме первых 5 строк (начиная с строки 6)
tail -n +5 file.txt

# Вывести всё кроме первых 100 байт
tail -c +100 file.txt

Режим отслеживания (follow)

-f отслеживает изменения файла в реальном времени:

# Отслеживать лог-файл
tail -f /var/log/syslog

# Выход: Ctrl+C

# Отслеживать с повторными попытками (если файл удалён и создан заново)
tail -F /var/log/app.log

Комбинация с head

# Строки с 10 по 20
head -n 20 file.txt | tail -n 10

# Строки с 5 по 15
tail -n +5 file.txt | head -n 11

Коды возврата

  • 0 — успешно
  • 1 — ошибка при чтении

less — постраничный просмотр

less позволяет просматривать файл постранично с возможностью прокрутки вперёд и назад.

Синопсис

less [ОПЦИЯ] [ФАЙЛ]...

Описание

Выводит файл на экран постранично, позволяя прокручивать содержимое вперёд и назад.

Опции

  • -N, --LINE-NUMBERS — показать номера строк
  • -S, --chop-long-lines — обрезать длинные строки (не переносить)
  • -R, --RAW-CONTROL-CHARS — выводить управляющие последовательности ANSI (цвета)
  • -F, --quit-if-one-screen — выйти, если содержимое помещается на одном экране
  • -G, --hilite-search — подсвечивать только последнее совпадение
  • -I, --IGNORE-CASE — игнорировать регистр при поиске
  • -M, --long-prompt — длинный prompt
  • -q, --quiet, --silent — не издавать звуковые сигналы
  • -X, --no-init — не очищать экран при выходе

Команды навигации

КлавишаОписание
Пробел, f, ^FВперёд на одну страницу
b, ^BНазад на одну страницу
↓, e, ^EВперёд на одну строку
↑, y, ^YНазад на одну строку
g, ^HomeВ начало файла
G, ^EndВ конец файла
/patternПоиск вперёд
?patternПоиск назад
nСледующее совпадение
NПредыдущее совпадение
q, ZZВыход
h, HСправка

Примеры

# Открыть файл для просмотра
less file.txt

# Открыть с номерами строк
less -N file.txt

# Открыть с поддержкой цветов
less -R file.txt

# Выйти, если файл маленький
less -F file.txt

# Открыть несколько файлов
less file1.txt file2.txt

# Перейти к строке 100
less +100 file.txt

# Искать с начала
less +/pattern file.txt

Команды поиска

# Искать вперёд
/pattern

# Искать назад
?pattern

# Следующее совпадение
n

# Предыдущее совпадение
N

# Искать без учёта регистра
-I

Коды возврата

  • 0 — успешно
  • 1 — ошибка при открытии файла
  • 2 — неверные опции командной строки

Сравнение команд просмотра

Практические примеры

Просмотр логов

# Последние 100 строк лога
tail -n 100 /var/log/syslog

# Отслеживать лог в реальном времени
tail -f /var/log/syslog

# Отслеживать с поиском
tail -f /var/log/syslog | grep --line-buffered error

# Просмотреть весь лог с навигацией
less /var/log/syslog

Анализ больших файлов

# Первые 20 строк
head -n 20 largefile.log

# Последние 20 строк
tail -n 20 largefile.log

# Строки с 100 по 120
head -n 120 largefile.log | tail -n 20

# Просмотр с навигацией
less largefile.log

Конвейеры

# Вывести первые 10 совпадений
grep "error" /var/log/syslog | head

# Вывести последние 5 совпадений
grep "error" /var/log/syslog | tail -n 5

# Нумеровать строки вывода
cat -n file.txt | head -n 20
