От основ Linux к мастерству Bash

Работа с файлами и директориями

В этом разделе рассматриваются основные команды для управления файлами и директориями: просмотр списка файлов, копирование, перемещение, создание и удаление.

ls — вывод списка файлов

ls выводит информацию о файлах в директории.

Синопсис

ls [ОПЦИЯ]... [ФАЙЛ]...

Описание

Выводит информацию о ФАЙЛах (текущей директории по умолчанию). Сортирует записи по алфавиту, если не указано иное.

Основные опции

  • -a, --all — не игнорировать записи, начинающиеся с .
  • -A, --almost-all — как -a, но без . и ..
  • -b, --escape — экранировать символы через \
  • -B, --ignore-backups — не выводить резервные копии (файлы ~)
  • -c — сортировать по времени изменения статуса
  • -C — выводить в колонках (по умолчанию для терминала)
  • --color[=КОГДА] — раскрашивать вывод: always, auto, never
  • -d, --directory — выводить информацию о директориях, а не их содержимом
  • -F, --classify — добавлять индикаторы типов файлов (/, *, @)
  • -f — не сортировать, выводить в порядке директории
  • -g — как -l, но без владельца
  • -G, --no-group — как -l, но без группы
  • -h, --human-readable — выводить размеры в читаемом формате (K, M, G)
  • --si — как -h, но использовать степени 1000
  • -i, --inode — выводить номер inode
  • -l — использовать длинный формат вывода
  • -L, --dereference — следовать за символическими ссылками
  • -m — выводить через запятую
  • -n, --numeric-uid-gid — выводить UID и GID вместо имён
  • -o — как -l, но без группы
  • -p — добавлять / к именам директорий
  • -r, --reverse — обратный порядок сортировки
  • -R, --recursive — рекурсивно выводить поддиректории
  • -s, --size — выводить размер каждого файла в блоках
  • -S — сортировать по размеру (наибольший первый)
  • --sort=СЛОВО — сортировать по: name, none, size, time, version, extension
  • -t — сортировать по времени изменения (новейший первый)
  • -u — сортировать по времени доступа
  • -U — не сортировать
  • -v — сортировать по версии
  • -w, --width=КОЛОНКИ — устанавливать ширину вывода
  • -x — выводить по колонкам, сортировать по горизонтали
  • -X — сортировать по расширению
  • -1 — выводить по одному файлу в строке
  • --help — вывести справку
  • --version — вывести версию

Примеры

# Вывести все файлы, включая скрытые
ls -a

# Длинный формат с читаемыми размерами
ls -lh

# Сортировать по времени (новейшие первые)
ls -lt

# Сортировать по размеру (наибольшие первые)
ls -lS

# Рекурсивный вывод
ls -R

# Вывести только директории
ls -ld */

# Раскрасить вывод
ls --color=always

# Вывести inode
ls -li

# Не выводить резервные копии
ls -B

# Вывести в колонках
ls -C

# Вывести по одному файлу в строке
ls -1

Форматы вывода

ОпцияФормат
-CКолонки (по умолчанию)
-lДлинный формат
-mЧерез запятую
-1По одному в строке
-xПо колонкам, горизонтально

Коды возврата

  • 0 — успешно
  • 1 — незначительные проблемы
  • 2 — серьёзные проблемы

cp — копирование файлов

cp копирует файлы и директории.

Синопсис

cp [ОПЦИЯ]... [-T] ИСТОЧНИК НАЗНАЧЕНИЕ
cp [ОПЦИЯ]... ИСТОЧНИК... НАЗНАЧЕНИЕ
cp [ОПЦИЯ]... -t НАЗНАЧЕНИЕ ИСТОЧНИК...

Основные опции

  • -a, --archive — рекурсивное копирование с сохранением всех атрибутов
  • -b — создавать резервные копии существующих файлов
  • -d — не следовать за ссылками, копировать ссылки
  • -f, --force — удалить существующий файл назначения
  • -i, --interactive — спрашивать перед перезаписью
  • -l, --link — создавать жёсткие ссылки вместо копирования
  • -L, --dereference — следовать за всеми ссылками
  • -n, --no-clobber — не перезаписывать существующие файлы
  • -P, --no-dereference — не следовать за ссылками
  • -p, --preserve — сохранить атрибуты (владелец, права, время)
  • -R, -r, --recursive — рекурсивное копирование директорий
  • -s, --symbolic-link — создавать символические ссылки
  • -t, --target-directory=ДИРЕКТОРИЯ — копировать в указанную директорию
  • -T, --no-target-directory — считать НАЗНАЧЕНИЕ обычным файлом
  • -u, --update — копировать только если файл новее
  • -v, --verbose — выводить информацию о копируемых файлах
  • -x, --one-file-system — не переходить на другие файловые системы

Примеры

# Копировать файл
cp file.txt backup.txt

# Копировать в другую директорию
cp file.txt /home/user/

# Копировать несколько файлов
cp file1.txt file2.txt /backup/

# Рекурсивное копирование директории
cp -r /home/user/docs /backup/

# Копировать с сохранением атрибутов
cp -a /home/user /backup/

# Интерактивное копирование (спрашивать перед перезаписью)
cp -i file.txt /existing/

# Не перезаписывать существующие файлы
cp -n file.txt /existing/

# Копировать только новые файлы
cp -u file.txt /backup/

# Создать резервную копию при перезаписи
cp -b file.txt /backup/

# Копировать с сохранением структуры директорий
cp --parents /home/user/file.txt /backup/
# Результат: /backup/home/user/file.txt

# Копировать символические ссылки как ссылки
cp -d link.txt /backup/

# Подробный вывод
cp -v file.txt /backup/

Особенности

Копирование директорий

Для копирования директорий необходимо использовать -R или -r:

# Ошибка: cp не копирует директории без -R
cp dir/ /backup/

# Правильно
cp -R dir/ /backup/

Сохранение атрибутов

-a (archive) сохраняет все атрибуты:

# Сохранить владельца, права, время
cp -a source/ dest/

# Сохранить только права и время
cp -p source/ dest/

Жёсткие и символические ссылки

# Копировать как жёсткую ссылку
cp -l file.txt link.txt

# Копировать как символическую ссылку
cp -s file.txt link.txt

# Следовать за ссылками
cp -L link.txt copy.txt

Коды возврата

  • 0 — успешно
  • 1 — ошибка при копировании

mv — перемещение и переименование файлов

mv перемещает или переименовывает файлы и директории.

Синопсис

mv [ОПЦИЯ]... [-T] ИСТОЧНИК НАЗНАЧЕНИЕ
mv [ОПЦИЯ]... ИСТОЧНИК... НАЗНАЧЕНИЕ
mv [ОПЦИЯ]... -t НАЗНАЧЕНИЕ ИСТОЧНИК...

Основные опции

  • -b, --backup — создавать резервные копии существующих файлов
  • -f, --force — не спрашивать перед перезаписью
  • -i, --interactive — спрашивать перед перезаписью
  • -n, --no-clobber — не перезаписывать существующие файлы
  • -S, --suffix=СУФФИКС — суффикс для резервных копий
  • -t, --target-directory=ДИРЕКТОРИЯ — переместить в указанную директорию
  • -T, --no-target-directory — считать НАЗНАЧЕНИЕ обычным файлом
  • -u, --update — перемещать только если файл новее
  • -v, --verbose — выводить информацию о перемещаемых файлах

Примеры

# Переименовать файл
mv oldname.txt newname.txt

# Переместить файл в другую директорию
mv file.txt /home/user/

# Переместить несколько файлов
mv file1.txt file2.txt /backup/

# Переместить директорию
mv /home/user/old_dir /home/user/new_dir

# Интерактивное перемещение (спрашивать перед перезаписью)
mv -i file.txt /existing/

# Не перезаписывать существующие файлы
mv -n file.txt /existing/

# Перемещать только новые файлы
mv -u file.txt /backup/

# Создать резервную копию при перезаписи
mv -b file.txt /backup/

# Подробный вывод
mv -v file.txt /backup/

# Переместить в целевую директорию
mv -t /backup/ file1.txt file2.txt

# Считать назначение файлом (не директорией)
mv -T file1.txt file2.txt

Особенности

Перемещение между файловыми системами

При перемещении между разными файловыми системами mv фактически копирует и удаляет:

# Перемещение на другой диск = копирование + удаление
mv /home/user/file.txt /mnt/external/

Переименование vs перемещение

# Переименование (в той же директории)
mv oldname.txt newname.txt

# Перемещение (в другую директорию)
mv file.txt /home/user/

# Переименование при перемещении
mv /home/user/old.txt /backup/new.txt

Резервное копирование

# Создать резервную копию с суффиксом ~
mv -b file.txt /backup/
# file.txt → file.txt~

# Нумерованные резервные копии
mv --backup=numbered file.txt /backup/

Коды возврата

  • 0 — успешно
  • 1 — ошибка при перемещении

mkdir — создание директорий

mkdir создаёт новые директории.

Синопсис

mkdir [ОПЦИЯ]... ИМЯ_ДИРЕКТОРИИ...

Основные опции

  • -m, --mode=РЕЖИМ — установить права доступа (как в chmod)
  • -p, --parents — создавать родительские директории; не выдавать ошибку, если директория существует
  • -v, --verbose — выводить информацию о создаваемых директориях
  • -Z, --context — установить контекст SELinux

Примеры

# Создать одну директорию
mkdir newdir

# Создать несколько директорий
mkdir dir1 dir2 dir3

# Создать директорию с правами 755
mkdir -m 755 newdir

# Создать директорию с правами только для владельца
mkdir -m 700 private

# Создать директорию с родительскими директориями
mkdir -p /home/user/project/src

# Создать и вывести информацию
mkdir -v newdir

# Не выдавать ошибку, если директория существует
mkdir -p existing_dir

Особенности

Создание вложенных директорий

Без -p mkdir не создаёт родительские директории:

# Ошибка: родительская директория не существует
mkdir /home/user/project/src

# Правильно: создать все директории
mkdir -p /home/user/project/src

Права доступа

По умолчанию mkdir создаёт директории с правами 777 - umask:

# Проверить umask
umask
# 0022

# Создать директорию (права будут 755)
mkdir newdir

# Создать с явными правами
mkdir -m 700 private

Массовое создание

# Создать несколько директорий
mkdir dir1 dir2 dir3

# Создать с поддиректориями
mkdir -p project/{src,docs,tests}

# Создать последовательность
mkdir dir_{1..10}

Коды возврата

  • 0 — успешно
  • 1 — ошибка при создании

rm — удаление файлов и директорий

rm удаляет файлы и директории.

Синопсис

rm [ОПЦИЯ]... ФАЙЛ...
rm [ОПЦИЯ]... -r --directory=ДИРЕКТОРИЯ...

Основные опции

  • -d, --directory — удалять пустые директории (как rmdir)
  • -f, --force — игнорировать несуществующие файлы, не спрашивать
  • -i — спрашивать перед удалением каждого файла
  • -I — спрашивать один раз перед удалением более 3 файлов
  • --interactive[=КОГДА] — режим опроса: always, never, once
  • -r, -R, --recursive — рекурсивное удаление директорий
  • -v, --verbose — выводить информацию об удаляемых файлах
  • --help — вывести справку
  • --version — вывести версию

Примеры

# Удалить файл
rm file.txt

# Удалить несколько файлов
rm file1.txt file2.txt

# Интерактивное удаление (спрашивать каждый файл)
rm -i file.txt

# Спрашивать один раз перед удалением многих файлов
rm -I *.txt

# Не спрашивать, игнорировать ошибки
rm -f file.txt

# Рекурсивное удаление директории
rm -r dir/

# Рекурсивное удаление с подтверждением
rm -ri dir/

# Удалить пустую директорию
rm -d empty_dir/

# Подробный вывод
rm -v file.txt

# Удалить все файлы в директории
rm /path/to/dir/*

# Удалить директорию со всем содержимым
rm -rf /path/to/dir/

Безопасность

rm — опасная команда, особенно с -rf:

# ОПАСНО: удаляет всё без вопросов
rm -rf /

# ОПАСНО: может удалить домашнюю директорию из-за опечатки
rm -rf /home/user/
rm -rf /home/user /*  # Пробел после / опасен!

# Безопаснее: использовать интерактивный режим
rm -ri dir/

Удаление директорий

# Ошибка: rm не удаляет директории без -r
rm dir/

# Правильно: рекурсивное удаление
rm -r dir/

# Только пустые директории (как rmdir)
rm -d dir/

Восстановление

Удалённые rm файлы не восстанавливаются:

# Файл удалён безвозвратно
rm important.txt

# Используйте корзину для безопасности
mv important.txt ~/.trash/

Коды возврата

  • 0 — успешно
  • 1 — ошибка при удалении

rmdir — удаление пустых директорий

rmdir удаляет только пустые директории.

Синопсис

rmdir [ОПЦИЯ]... ИМЯ_ДИРЕКТОРИИ...

Основные опции

  • -p, --parents — удалять родительские директории после удаления дочерних
  • -v, --verbose — выводить информацию об удаляемых директориях

Примеры

# Удалить пустую директорию
rmdir empty_dir

# Удалить несколько пустых директорий
rmdir dir1 dir2 dir3

# Удалить с выводом информации
rmdir -v empty_dir

# Удалить директорию и пустые родительские
rmdir -p /home/user/project/src/docs
# Удаляет: docs, затем src, затем project (если пустые)

Особенности

Только пустые директории

rmdir удаляет только пустые директории:

# Ошибка: директория не пуста
rmdir dir/
# rmdir: failed to remove 'dir/': Directory not empty

# Правильно: сначала удалить содержимое
rm -r dir/

# Или использовать rm с -d для пустых
rm -d dir/

Удаление цепочки директорий

-p удаляет родительские директории, если они становятся пустыми:

# Создать вложенные директории
mkdir -p a/b/c/d

# Удалить с конца
rmdir -p a/b/c/d
# Удаляет: d → c → b → a (если все пустые)

Коды возврата

  • 0 — успешно
  • 1 — ошибка при удалении

Практические примеры

Работа с проектами

# Создать структуру проекта
mkdir -p project/{src,docs,tests}

# Скопировать проект
cp -r project project_backup

# Переместить в архив
mv project /archive/

# Удалить временные файлы
rm -rf project/tmp/

Резервное копирование

# Создать резервную копию с датой
cp -a important/ important_$(date +%Y%m%d)/

# Переместить старую резервную копию
mv old_backup/ archive/

# Удалить старые резервные копии
rm -rf archive/backup_2023*/

Безопасное удаление

# Интерактивное удаление
rm -i *.tmp

# Сначала просмотреть, потом удалить
ls *.tmp
rm *.tmp

# Использовать корзину вместо rm
mv unwanted.txt ~/.trash/
