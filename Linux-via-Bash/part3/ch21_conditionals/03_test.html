От основ Linux к мастерству Bash

test и [ ]

test и [ ] проверяют условия и возвращают статус выхода.

Синтаксис

test expression
[ expression ]

Важно: Пробелы вокруг выражения обязательны!

[ $x -eq 5 ]   # Правильно
[$x -eq 5]     # ОШИБКА!

Строковые операторы

ОператорОписание
-z stringПустая строка
-n stringНепустая строка
string1 = string2Равны
string1 != string2Не равны

Примеры

# Пустая строка
if [ -z "$var" ]; then
    echo "Пусто"
fi

# Непустая
if [ -n "$var" ]; then
    echo "Не пусто: $var"
fi

# Равенство
if [ "$name" = "John" ]; then
    echo "Hello John"
fi

# Неравенство
if [ "$name" != "admin" ]; then
    echo "Not admin"
fi

Числовые операторы

ОператорОписание
arg1 -eq arg2Равно
arg1 -ne arg2Не равно
arg1 -lt arg2Меньше
arg1 -le arg2Меньше или равно
arg1 -gt arg2Больше
arg1 -ge arg2Больше или равно

Примеры

if [ $x -eq 5 ]; then
    echo "x равно 5"
fi

if [ $x -ne 0 ]; then
    echo "x не ноль"
fi

if [ $x -lt 10 ]; then
    echo "x меньше 10"
fi

if [ $x -ge 18 ]; then
    echo "Совершеннолетний"
fi

Операторы с файлами

ОператорОписание
-a fileФайл существует
-d fileДиректория
-e fileФайл существует
-f fileОбычный файл
-r fileЧитаемый
-s fileРазмер > 0
-w fileЗаписываемый
-x fileИсполняемый
-L fileСимволическая ссылка

Примеры

# Обычный файл
if [ -f file.txt ]; then
    echo "Это файл"
fi

# Директория
if [ -d /tmp ]; then
    echo "/tmp существует"
fi

# Исполняемый
if [ -x script.sh ]; then
    ./script.sh
fi

# Пустой файл
if [ ! -s file.txt ]; then
    echo "Файл пуст"
fi

Логические операторы

ОператорОписание
! exprНЕ
expr1 -a expr2И (and)
expr1 -o expr2ИЛИ (or)

Примеры

# И
if [ -f file.txt -a -r file.txt ]; then
    cat file.txt
fi

# ИЛИ
if [ -z $var1 -o -z $var2 ]; then
    echo "Одна переменная пуста"
fi

# НЕ
if [ ! -f file.txt ]; then
    echo "Файл не существует"
fi

Сравнение файлов

ОператорОписание
file1 -nt file2file1 новее
file1 -ot file2file1 старше
file1 -ef file2Одинаковые inode

Примеры

if [ script.sh -nt script.sh.bak ]; then
    echo "Скрипт новее"
fi

if [ file1 -ef file2 ]; then
    echo "Один и тот же файл"
fi

Примеры использования

Проверка аргументов

if [ $# -eq 0 ]; then
    echo "Нет аргументов"
    exit 1
fi

if [ $# -gt 3 ]; then
    echo "Слишком много аргументов"
    exit 1
fi

Проверка диапазона

if [ $age -ge 18 ] && [ $age -le 65 ]; then
    echo "Трудоспособный"
fi

Проверка существования команды

if [ -x /usr/bin/git ]; then
    echo "git установлен"
fi

Цикл с проверкой

while [ $i -lt 10 ]; do
    echo $i
    i=$((i+1))
done

Статус выхода

  • 0 — условие истинно
  • 1 — условие ложно
  • >1 — ошибка

test vs [ ]

test $x -eq 5    # test
[ $x -eq 5 ]     # [ ] - то же самое

[ ] — это синоним test, но требует ] в конце.

Распространённые ошибки

Пропущенные пробелы

# НЕПРАВИЛЬНО:
[$x -eq 5]
[ $x -eq 5]
[$x -eq 5 ]

# ПРАВИЛЬНО:
[ $x -eq 5 ]

Неэкранированные переменные

# НЕПРАВИЛЬНО (ошибка если пусто):
if [ $var = "value" ]; then

# ПРАВИЛЬНО:
if [ "$var" = "value" ]; then

Неправильные операторы

# НЕПРАВИЛЬНО (== вместо =):
[ $x == 5 ]

# ПРАВИЛЬНО:
[ $x = 5 ]

# НЕПРАВИЛЬНО (< > без экранирования):
[ $x < $y ]

# ПРАВИЛЬНО:
[ $x -lt $y ]
