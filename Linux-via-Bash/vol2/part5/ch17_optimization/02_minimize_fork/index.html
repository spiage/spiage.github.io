    Минимизация fork/exec¶

    Каждая внешняя команда создаёт новый процесс.

    Стоимость fork/exec¶

    # Время выполнения
time ls           # ~0.002s (fork + exec)
time echo test    # ~0.000s (builtin)

    Накладные расходы: - fork() — копирование процесса - exec() — загрузка программы - wait() — ожидание завершения

    Избегание fork¶

    builtin вместо external¶

    # Плохо (3 fork)
cat file | grep pattern | wc -l

# Лучше (2 fork)
grep pattern file | wc -l

# Ещё лучше (1 fork)
grep -c pattern file

# Оптимально (1 fork)
# builtin для grep нет, но это эффективный способ
grep -c pattern file

    В циклах¶

    # Плохо (1000 fork)
for i in {1..1000}; do
    date +%s
done

# Хорошо (0 fork)
for i in {1..1000}; do
    echo $i
done

    Примечание: {1..1000} — расширение bash. В dash используйте seq 1 1000 (внешняя команда, 1 fork) или POSIX-цикл i=1; while [ $i -le 1000 ]; do ...; i=$((i+1)); done.

    Группировка команд¶

    # Плохо (3 fork)
ls /etc > out.txt
ls /var >> out.txt
ls /tmp >> out.txt

# Хорошо (1 fork)
{
    ls /etc
    ls /var
    ls /tmp
} > out.txt

    Практические примеры¶

    Чтение файла¶

    # Плохо (fork на cat)
cat file | while read line; do
    echo "$line"
done

# Хорошо (0 fork)
while read line; do
    echo "$line"
done < file

    Сумма чисел¶

    # Плохо (awk fork)
cat file | awk '{sum+=$1} END {print sum}'

# Хорошо (1 fork)
awk '{sum+=$1} END {print sum}' file

    Массовое удаление¶

    # Плохо (fork на каждый rm)
for f in *.tmp; do
    rm "$f"
done

# Хорошо (1 fork)
rm *.tmp

# Или (1 fork)
find . -name "*.tmp" | xargs rm

    subshell и fork¶

    # subshell создаёт fork
(
    cd /tmp
    pwd
)

# builtin не создаёт
{
    cd /tmp
    pwd
}
