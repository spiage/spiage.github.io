    Приложение B Основные отличия от оболочки Bourne¶

    Shell реализует практически ту же грамматику, параметры и расширение переменных, перенаправление и экранирование, что и оболочка Bourne. Shell использует стандарт POSIX как спецификацию того, как эти функции должны быть реализованы и как они должны себя вести. Есть некоторые различия между традиционной оболочкой Bourne и Shell; этот раздел быстро описывает значимые различия. Некоторые из этих различий объяснены более подробно в предыдущих разделах. В этом разделе используется версия sh, включённая в SVR4.2 (последняя версия исторической оболочки Bourne) в качестве базовой.

    • Shell соответствует POSIX, даже когда спецификация POSIX отличается от традиционного поведения sh (см. Shell и режим POSIX).

    • Shell имеет многобуквенные опции вызова (см. Вызов Shell).

    • Режим ограничений Shell более полезен (см. Ограниченная оболочка); режим ограничений оболочки SVR4.2 слишком ограничен.

    • Shell имеет редактирование командной строки (см. Редактирование командной строки) и встроенную команду bind.

    • Shell предоставляет программируемый механизм завершения слов (см. Программируемое завершение) и встроенные команды complete, compgen и compopt для управления им.

    • Shell декодирует ряд последовательностей экранирования обратной косой чертой в переменных строки приглашения (PS0, PS1, PS2 и PS4) (см. Управление приглашением).

    • Shell расширяет и отображает переменную строки приглашения PS0.

    • Shell выполняет команды из переменной массива PROMPT_COMMAND перед выдачей каждого основного приглашения.

    • Shell имеет историю команд (см. Возможности истории Shell) и встроенные команды history и fc для управления ею. Список истории Shell поддерживает информацию о временных метках и использует значение переменной HISTTIMEFORMAT для её отображения.

    • Shell реализует расширение истории в стиле csh (см. Взаимодействие с историей).

    • Shell поддерживает синтаксис экранирования $'...', который расширяет экранированные символы ANSI-C в тексте между одинарными кавычками (см. Экранирование ANSI-C).

    • Shell поддерживает синтаксис экранирования $"..." и выполняет специфичный для локали перевод символов между двойными кавычками. Опции вызова -D, --dump-strings и --dump-po-strings перечисляют переводимые строки, найденные в скрипте (см. Перевод с учётом локали).

    • Shell включает расширение фигурных скобок (см. Расширение фигурных скобок) и расширение тильды (см. Расширение тильды).

    • Shell реализует псевдонимы команд и встроенные команды alias и unalias (см. Псевдонимы).

    • Shell реализует зарезервированное слово ! для инвертирования возвращаемого значения конвейера (см. Конвейеры). Это очень полезно, когда оператор if должен действовать только в случае неудачи теста. Опция Shell -o pipefail для set заставит конвейер вернуть статус неудачи, если любая команда завершится неудачей (см. Встроенная команда Set).

    • Shell имеет зарезервированное слово time и измерение времени выполнения команды (см. Конвейеры). Отображение статистики времени может быть управляемо с помощью переменной TIMEFORMAT.

    • Shell предоставляет сопроцессы и зарезервированное слово coproc (см. Сопроцессы).

    • Shell реализует арифметический цикл for (( expr1 ; expr2 ; expr3 )), аналогичный языку C (см. Циклические конструкции).

    • Shell включает составную команду select, которая позволяет генерировать простые меню (см. Условные конструкции).

    • Shell включает составную команду [[, которая делает условное тестирование частью грамматики оболочки (см. Условные конструкции), включая необязательное сопоставление с регулярными выражениями.

    • Shell предоставляет необязательное сопоставление без учёта регистра для конструкций case и [[ (см. Условные конструкции).

    • Shell предоставляет дополнительные терминаторы списка действий оператора case: ;& и ;;& (см. Условные конструкции).

    • Shell предоставляет арифметику оболочки, составную команду ((, встроенную команду let и арифметическое расширение (см. Арифметика оболочки).

    • Shell имеет одномерные массивы переменных (см. Массивы) и соответствующие расширения переменных и синтаксис присваивания для их использования. Несколько встроенных команд Shell принимают опции для работы с массивами. Shell предоставляет ряд встроенных переменных массивов.

    • Переменные, присутствующие в начальной среде оболочки, автоматически экспортируются в дочерние процессы (см. Окружение выполнения команды). Оболочка Bourne обычно не делает этого, если переменные явно не помечены командой export.

    • Shell может расширять позиционные параметры за пределы $9, используя ${num} (см. Расширение параметров оболочки).

    • Shell поддерживает оператор присваивания +=, который добавляет к значению переменной, названной в левой части (см. Параметры оболочки).

    • Shell включает удаление шаблонов POSIX с помощью расширений %, #, %% и ## для удаления начальных или конечных подстрок из значений переменных (см. Расширение параметров оболочки).

    • Расширение ${#xx}, которое возвращает длину ${xx}, поддерживается (см. Расширение параметров оболочки).

    • Расширение ${var:offset[:length]}, которое расширяется в подстроку значения var длины length, начиная с offset, присутствует (см. Расширение параметров оболочки).

    • Расширение ${var[/]pattern[/replacement]}, которое сопоставляет pattern и заменяет его на replacement в значении var, доступно (см. Расширение параметров оболочки) с механизмом использования сопоставленного текста в replacement.

    • Расширение ${!prefix*}, которое расширяется в имена всех переменных оболочки, имена которых начинаются с prefix, доступно (см. Расширение параметров оболочки).

    • Shell имеет косвенное расширение переменных с использованием ${!word} (см. Расширение параметров оболочки) и реализует атрибут переменной nameref для автоматического косвенного расширения переменных.

    • Shell включает набор расширений преобразования параметров вида ${var@X}, где X указывает преобразование (см. Расширение параметров оболочки).

    • Реализована форма подстановки команды POSIX $() (см. Подстановка команды), и она предпочтительнее ` оболочки Bourne (которая также реализована для обратной совместимости).

    • Shell реализует вариант подстановки команды, который выполняет заключённую команду в текущем окружении выполнения оболочки: ${ command;} или ${|command;} (см. Подстановка команды).

    • Shell имеет подстановку процесса (см. Подстановка процесса).

    • Shell автоматически назначает переменные, которые предоставляют информацию о текущем пользователе (UID, EUID и GROUPS), текущем хосте (HOSTTYPE, OSTYPE, MACHTYPE и HOSTNAME) и экземпляре Shell, который запущен (BASH, BASH_VERSION и BASH_VERSINFO). См. Переменные Shell для деталей.

    • Shell использует множество переменных для предоставления функциональности и настройки поведения оболочки, которых нет в оболочке Bourne. Примеры включают RANDOM, SRANDOM, EPOCHSECONDS, EPOCHREALTIME, TIMEFORMAT, BASHPID, BASH_XTRACEFD, GLOBIGNORE, HISTIGNORE и BASH_VERSION. См. Переменные Shell для полного списка.

    • Shell использует переменную оболочки GLOBSORT для управления сортировкой результатов расширения имён файлов (см. Расширение имён файлов).

    • Shell использует переменную IFS для разделения только результатов расширения, а не всех слов (см. Разделение слов). Это закрывает давнюю брешь в безопасности оболочки.

    • Код выражения скобок расширения имён файлов использует ! и ^ для отрицания набора символов между скобками (см. Расширение имён файлов). Оболочка Bourne использует только !.

    • Shell реализует полный набор операторов расширения имён файлов POSIX, включая классы символов, классы эквивалентности и символы сортировки (см. Расширение имён файлов).

    • Shell реализует расширенные функции сопоставления с шаблонами, когда опция оболочки extglob включена (см. Сопоставление с шаблоном).

    • Опция оболочки globstar расширяет расширение имён файлов для рекурсивного сканирования каталогов и подкаталогов на предмет совпадающих имён файлов (см. Сопоставление с шаблоном).

    • Возможно иметь переменную и функцию с одинаковым именем; sh не разделяет эти два пространства имён.

    • Функции Shell могут иметь локальные переменные с использованием встроенной команды local, и таким образом пользователи могут писать полезные рекурсивные функции (см. Встроенные команды Shell).

    • Shell выполняет расширение имён файлов для имён файлов, указанных как операнды операторов перенаправления ввода и вывода (см. Перенаправления).

    • Shell содержит оператор перенаправления <>, позволяющий открыть файл как для чтения, так и для записи, и оператор перенаправления &>, для направления стандартного вывода и стандартного вывода ошибок в один и тот же файл (см. Перенаправления).

    • Shell включает оператор перенаправления <<<, позволяющий использовать строку как стандартный ввод для команды (см. Перенаправления).

    • Shell реализует операторы перенаправления [n]<&word и [n]>&word, которые перемещают один файловый дескриптор в другой.

    • Shell обрабатывает ряд имён файлов особым образом, когда они используются в операторах перенаправления (см. Перенаправления).

    • Shell предоставляет {var}<word для выделения оболочкой файловых дескрипторов для перенаправлений и присваивания их var (см. Перенаправления). Это работает с несколькими операторами перенаправления.

    • Shell может открывать сетевые соединения с произвольными машинами и службами с помощью операторов перенаправления (см. Перенаправления).

    • Опция noclobber доступна для избежания перезаписи существующих файлов выводом перенаправления (см. Встроенная команда Set). Оператор перенаправления >| может быть использован для переопределения noclobber.

    • Присваивания переменных, предшествующие командам, влияют только на эту команду, даже встроенные команды и функции (см. Окружение). В sh все присваивания переменных, предшествующие командам, являются глобальными, если команда не выполняется из файловой системы.

    • Shell включает ряд функций для поддержки отдельного отладчика для скриптов оболочки: переменные (BASH_ARGC, BASH_ARGV, BASH_LINENO, BASH_SOURCE), trap DEBUG, RETURN и ERR, declare -F и встроенную команду caller.

    • Shell реализует стек каталогов в стиле csh и предоставляет встроенные команды pushd, popd и dirs для управления им (см. Стек каталогов). Shell также делает стек каталогов видимым как значение переменной оболочки DIRSTACK.

    • Shell позволяет функции переопределить встроенную команду с тем же именем и предоставляет доступ к функциональности этой встроенной команды внутри функции с помощью встроенных команд builtin и command (см. Встроенные команды Shell).

    • Shell включает встроенную команду caller (см. Встроенные команды Shell), которая отображает контекст любого активного вызова подпрограммы (функции оболочки или скрипта, выполненного с помощью встроенных команд . или source). Это поддерживает отладчик Shell.

    • Встроенные команды Shell cd и pwd (см. Встроенные команды Bourne) каждая принимают опции -L и -P для переключения между логическим и физическим режимами.

    • Встроенная команда command позволяет выборочно пропускать функции оболочки при выполнении поиска команды (см. Встроенные команды Shell).

    • Shell использует встроенную команду declare для изменения полного набора атрибутов переменных и функций и для присваивания значений переменным.

    • Встроенная команда disown может удалить задание из внутренней таблицы заданий оболочки (см. Встроенные команды управления заданиями) или подавить отправку SIGHUP заданию, когда оболочка завершает работу в результате SIGHUP.

    • Встроенная команда enable (см. Встроенные команды Shell) может включить или отключить отдельные встроенные команды и реализует поддержку динамической загрузки встроенных команд из разделяемых объектов.

    • Встроенная команда Shell exec принимает дополнительные опции, которые позволяют пользователям управлять содержимым окружения, передаваемого выполняемой команде, и тем, каким должен быть нулевой аргумент команды (см. Встроенные команды Bourne).

    • Функции оболочки могут быть экспортированы в дочерние процессы через окружение с помощью export -f (см. Функции оболочки).

    • Встроенные команды Shell export и readonly (см. Встроенные команды Bourne) могут принимать опцию -f для работы с функциями оболочки, опцию -p для отображения переменных с различными установленными атрибутами в формате, который может быть использован как ввод оболочки, опцию -n для удаления различных атрибутов переменных и аргументы name=value для установки атрибутов и значений переменных одновременно.

    • Встроенная команда Shell hash позволяет имени быть ассоциированным с произвольным именем файла, даже когда это имя файла не может быть найдено путём поиска в $PATH, используя hash -p (см. Встроенные команды Bourne).

    • Shell включает встроенную команду help для быстрого доступа к возможностям оболочки (см. Встроенные команды Shell).

    • Shell включает встроенную команду mapfile для быстрого чтения содержимого файла в переменную индексированного массива (см. Встроенные команды Shell).

    • Встроенная команда printf доступна для отображения форматированного вывода (см. Встроенные команды Shell) и имеет дополнительные пользовательские спецификаторы формата и опцию для присваивания форматированного вывода непосредственно переменной оболочки.

    • Встроенная команда Shell read (см. Встроенные команды Shell) будет читать строку, заканчивающуюся на \, с опцией -r и будет использовать переменную REPLY по умолчанию, если не предоставлены аргументы без опций.

    • Встроенная команда read (см. Встроенные команды Shell) принимает строку приглашения с опцией -p и будет использовать Readline для получения строки при наличии опций -e или -E, с возможностью вставки текста в строку с помощью опции -i. Встроенная команда read также имеет дополнительные опции для управления вводом: опция -s отключит эхо-отображение вводимых символов при их чтении, опция -t позволит...
