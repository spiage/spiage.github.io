Блокирующие и неблокирующие системные вызовы

Системные вызовы могут блокировать процесс до завершения операции.

Блокирующие вызовы

Блокирующий вызов не возвращает управление пока операция не завершится.

Процесс:  ━━━━━━━━●━━━━━━━━
                     ↑
                 Блокировка
                 (ждёт I/O)

Примеры блокирующих операций

ВызовКогда блокируетСостояние
read(fd, ...)Нет данных в fdS (sleep)
write(fd, ...)Буфер полонD (disk sleep)
accept()Нет подключенийS
connect()Соединение не установленоS
wait()Нет завершённых детейS
open()Файл на медленном устройствеD
fcntl(..., F_SETLK)Блокировка занятаS

Проверка состояния

# Посмотреть состояния процессов
ps aux | awk '{print $8, $11}' | sort | uniq -c

# Состояния:
# R — running/running on CPU
# S — interruptible sleep (ждёт событие)
# D — uninterruptible sleep (ждёт I/O)
# Z — zombie (завершён, не убран)
# T — stopped (сигналом)

Состояния процесса

S (Interruptible Sleep)

Процесс ждёт событие и может быть прерван сигналом:

# Большинство спящих процессов в S
ps aux | grep ' S ' | head -5

# Пример:
# USER   PID  S  CMD
# user  5678  S  bash
# user  5679  S  ssh user@host

Почему прерываемый:

  • Ждёт ввод с терминала
  • Ждёт данные из сети
  • Ждёт завершение дочернего процесса

D (Uninterruptible Sleep)

Процесс ждёт I/O и не может быть прерван:

# Найти процессы в D
ps aux | grep ' D '

# Пример:
# USER   PID  S  CMD
# root   123  D  [kworker/u4:0]

Причины D-состояния:

  • Ожидание ответа от диска
  • Ожидание сетевого ресурса
  • Блокировка ядра

Проблема: kill -9 не работает!

# kill игнорируется
kill -9 <pid>  # Ничего не происходит

# Решение: исправить I/O проблему или перезагрузиться

Пример: блокирующий read

# cat без аргументов ждёт ввода с терминала
cat
# Процесс в состоянии S, ждёт данные с stdin

# В strace видно:
strace cat
# read(0, ...
# (завис, ждёт ввод)

В скрипте:

#!/bin/bash

# Может зависнуть навсегда
read user_input

# С таймаутом
read -t 10 user_input || echo "Timeout!"

Пример: блокирующий write

# Запись в pipe без читателя
mkfifo /tmp/test_pipe
cat /dev/zero > /tmp/test_pipe
# Блокируется после заполнения буфера pipe (64KB)

# strace показывает:
strace cat /dev/zero > /tmp/test_pipe 2>&1 | grep write
# write(1, "\0\0\0...", 65536) = 65536
# write(1, "\0\0\0...", 65536)  # (завис — буфер полон)

Неблокирующий режим

Можно установить флаг O_NONBLOCK:

// Блокирующий (по умолчанию)
int fd = open("file", O_RDONLY);
read(fd, buf, size);  // Ждёт данные

// Неблокирующий
int fd = open("file", O_RDONLY | O_NONBLOCK);
read(fd, buf, size);  // Возвращает EAGAIN если нет данных

В bash

# timeout использует неблокирующее ожидание
timeout 5 sleep 100  # Завершится через 5 секунд

# read с таймаутом
read -t 5 input  # Вернётся через 5 секунд

Влияние на скрипты

Проблема: зависший скрипт

#!/bin/bash

# Скрипт может зависнуть на блокирующем вызове
cat | grep pattern  # Ждёт ввод с stdin бесконечно

# Решение: таймаут
timeout 10 cat | grep pattern

Проблема: pipe без читателя

#!/bin/bash

# Запись в pipe без читателя завершается ошибкой
echo "data" | { read x; exit 1; }
echo "more data"  # SIGPIPE, write вернёт EPIPE

# Обработка
set -o pipefail
echo "data" | { read x; exit 1; } || echo "Pipe failed"

Решение: обработка SIGPIPE

#!/bin/bash

trap 'echo "Pipe broken"; exit 1' PIPE

echo "data" | { read x; exit 1; }
echo "more data"  # Обработчик вызовется

Проблема: медленный диск

#!/bin/bash

# Чтение с медленного диска может блокировать
dd if=/dev/sdb of=/dev/null bs=1M

# Решение: timeout
timeout 60 dd if=/dev/sdb of=/dev/null bs=1M || \
    echo "Disk too slow or failed"

Диагностика блокировок

Найти зависшие процессы

# Процессы в состоянии D
ps aux | grep -E '^[^ ]+ +[0-9]+ .* [SD]'

# Или через state
ps -eo pid,state,comm | grep -E '^[[:space:]]*[0-9]+[[:space:]]+[SD]'

Посмотреть, что ждёт процесс

# Имя функции ядра, где спит
cat /proc/<pid>/wchan

# Пример:
# cat /proc/1234/wchan
# unix_stream_data_ready  # Ждёт данные из сокета

Посмотреть открытые файлы

# FD процесса
lsof -p <pid>

# Или через /proc
ls -l /proc/<pid>/fd/

Трассировка зависшего процесса

# Прикрепить strace
strace -p <pid>

# Покажет текущий syscall
# read(0, ...  # (ждёт ввод)

Практические рекомендации

1. Используйте таймауты

#!/bin/bash

# Сетевые операции
timeout 30 curl http://example.com

# Чтение ввода
read -t 10 user_input

# Команды
timeout 60 long_running_command

2. Проверяйте наличие читателя

#!/bin/bash

# Перед записью в pipe
if [[ ! -p /tmp/mypipe ]]; then
    echo "Pipe does not exist"
    exit 1
fi

# Или с обработкой ошибки
echo "data" > /tmp/mypipe 2>/dev/null || \
    echo "No reader, pipe blocked"

3. Обрабатывайте SIGPIPE

#!/bin/bash

trap 'echo "SIGPIPE received"; exit 1' PIPE

# Конвейер
producer | consumer

4. Мониторьте D-состояния

#!/bin/bash

# Проверка на D-процессы
D_COUNT=$(ps aux | grep -c ' D ')

if [[ $D_COUNT -gt 0 ]]; then
    echo "Warning: $D_COUNT processes in D state"
    ps aux | grep ' D '
fi

5. Неблокирующий ввод в скриптах

#!/bin/bash

# Проверка ввода без блокировки
if read -t 0 -n 1 input; then
    echo "Got input: $input"
else
    echo "No input available"
fi

Примеры из практики

Зависший mount

# NFS сервер не отвечает
mount -t nfs server:/export /mnt

# Процесс в D-состоянии
ps aux | grep mount

# Нельзя убить
kill -9 <pid>  # Игнорируется

# Решение: перезагрузить или исправить NFS

Блокировка на сокете

# netcat ждёт подключения
nc -l 8080

# В другом терминале:
ps aux | grep nc
# user  1234  S  nc -l 8080  # S состояние

# Подключиться для разблокировки
nc localhost 8080

Конвейер с зомби

#!/bin/bash

# Создать зомби
( exit 0 ) &

# Родитель не делает wait
sleep 100 &

# Проверить зомби
ps aux | grep Z
