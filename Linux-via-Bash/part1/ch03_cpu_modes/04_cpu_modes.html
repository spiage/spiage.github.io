Инструкции переключения режимов процессора

Инструкции для перехода между user mode и kernel mode.

AMD64: syscall / sysret

syscall

syscall — быстрая инструкция для перехода из user mode в kernel mode.

; User mode вызывает syscall
syscall         ; RCX ← RIP       ; Сохранить адрес возврата
                ; R11 ← RFLAGS    ; Сохранить флаги
                ; RIP ← IA32_LSTAR; Переход в Ring 0
                ; CS ← kernel_CS  ; Сегмент ядра
                ; SS ← kernel_SS  ; Сегмент ядра

Что происходит:

  1. Сохраняется адрес возврата в RCX
  2. Сохраняются флаги в R11
  3. RIP загружается из MSR IA32_LSTAR (адрес обработчика)
  4. Процессор переключается в Ring 0

sysret

sysret — возврат из kernel mode в user mode.

; Ядро возвращает управление
sysret          ; RIP ← RCX       ; Восстановить адрес
                ; RFLAGS ← R11    ; Восстановить флаги
                ; CS ← user_CS    ; Сегмент пользователя
                ; SS ← user_SS    ; Сегмент пользователя

Почему int 0x80 медленнее:

int 0x80:
1. Сохранить состояние
2. Найти вектор прерывания в IDT
3. Проверить права
4. Переключить стек
5. Выполнить обработчик

syscall:
1. Сохранить RCX, R11
2. Загрузить RIP из MSR
3. Переключить Ring

ARM64: svc / eret

svc (Supervisor Call)

svc — переход из EL0 (user) в EL1 (kernel).

; User mode (EL0) вызывает svc
svc #0          ; Переход в EL1
                ; SPSR_EL1 ← PSTATE  ; Сохранить состояние
                ; ELR_EL1 ← PC       ; Сохранить адрес
                ; PC ← VBAR_EL1      ; Вектор прерываний

eret (Exception Return)

eret — возврат из исключения.

; Ядро возвращает управление
eret            ; PSTATE ← SPSR_EL1  ; Восстановить состояние
                ; PC ← ELR_EL1       ; Восстановить адрес

RISC-V: ecall / mret / sret

ecall (Environment Call)

ecall — вызов исключения для перехода в старший режим.

; User mode вызывает ecall
ecall           ; Переход в S-mode
                ; xRA ← PC+4       ; Адрес возврата
                ; PC ← stvec       ; Вектор исключений

sret / mret

sret — возврат из S-mode в U-mode.

; Ядро возвращает управление
sret            ; PC ← xEPC        ; Восстановить адрес
                ; xSTATUS ← xSTATUS ; Восстановить статус

mret — возврат из M-mode (firmware).

Что происходит при запросе к ядру

sequenceDiagram
    participant U as User Mode (Ring 3)
    participant K as Kernel Mode (Ring 0)
    
    U->>U: Выполнение кода
    U->>K: syscall / svc / ecall
    Note over K: Сохранение регистров<br/>Проверка аргументов<br/>Выполнение операции
    K->>U: Возврат результата
    Note over U: Продолжение выполнения

Таблица запросов к ядру

Ядро хранит таблицу указателей на обработчики:

// Упрощённо (arch/x86/entry/syscalls/syscall_64.tbl)
static const sys_call_ptr_t sys_call_table[__NR_syscalls] = {
    [__NR_read] = sys_read,
    [__NR_write] = sys_write,
    [__NR_open] = sys_open,
    [__NR_close] = sys_close,
    [__NR_fork] = sys_fork,
    [__NR_execve] = sys_execve,
    [__NR_exit] = sys_exit,
    [__NR_kill] = sys_kill,
    // ...
};

При syscall:

  1. Номер syscall загружается в RAX
  2. Ядро читает sys_call_table[RAX]
  3. Выполняется соответствующий обработчик

Практическое наблюдение

strace

# Трассировка конкретного syscall
strace -e trace=write bash -c 'echo hello'
# write(1, "hello\n", 7) = 7

# Трассировка с номерами
strace -e trace=write -v bash -c 'echo hello'
# 1 write(1, "hello\n", 7) = 7

ausyscall

# Номер по имени
ausyscall write
# 1

# Имя по номеру
ausyscall 1
# write

/proc

# Таблица syscall текущего ядра
cat /proc/sys/kernel/sysctl_writes_strict

# Или через таблицу
grep -E '^#define __NR_' /usr/include/asm/unistd_64.h | head -10
# #define __NR_read 0
# #define __NR_write 1
# #define __NR_open 2
