Inode и VFS

Внутренняя структура файловой системы Linux.

Inode (индексный узел)

Inode — структура данных, хранящая метаданные файла.

Что хранится в inode

ПолеОписание
Номер inodeУникальный идентификатор
Тип файлаОбычный, директория, устройство, и т.д.
Права доступаrwx для владельца/группы/остальных
ВладелецUID владельца
ГруппаGID владельца
РазмерРазмер файла в байтах
Временные меткиatime, mtime, ctime
СсылкиКоличество жёстких ссылок
Указатели на блокиГде данные хранятся на диске

Что НЕ хранится в inode

  • Имя файла — хранится в директории
  • Путь к файлу — вычисляется при обходе

Просмотр inode

# Номер inode файла
ls -i file.txt
# 1234567 file.txt

# Подробная информация
stat file.txt
#   Device: 801h/2049d  Inode: 1234567
#   Size: 1234 bytes   Links: 1
#   Access: (0644/-rw-r--r--)  Uid: 1000/user  Gid: 1000/user
#   Access: 2026-03-14 10:00:00
#   Modify: 2026-03-14 09:00:00
#   Change: 2026-03-14 09:00:00

Жёсткие ссылки

Жёсткая ссылка — несколько имён файлов указывают на один inode.

# Создать жёсткую ссылку
ln file.txt hardlink.txt

# Оба имени указывают на один inode
ls -i file.txt hardlink.txt
# 1234567 file.txt
# 1234567 hardlink.txt

# Счётчик ссылок увеличился
stat file.txt | grep Links
# Links: 2

Важно:

  • Нельзя создать жёсткую ссылку на другую файловую систему
  • Нельзя создать жёсткую ссылку на директорию (защита от циклов)
  • Файл удаляется только когда счётчик ссылок = 0

Пример удаления:

# Создать файл и ссылку
echo "data" > file.txt
ln file.txt hardlink.txt

# Удалить оригинал
rm file.txt

# Данные всё ещё доступны через ссылку!
cat hardlink.txt
# data

# Удалить ссылку — только теперь данные удаляются
rm hardlink.txt

Символические ссылки

Символическая ссылка — специальный файл, содержащий путь к другому файлу.

# Создать символическую ссылку
ln -s file.txt symlink.txt

# Ссылка имеет свой inode
ls -i file.txt symlink.txt
# 1234567 file.txt
# 7654321 symlink.txt

# Ссылка показывает куда указывает
ls -l symlink.txt
# lrwxrwxrwx 1 user user 8 Mar 14 10:00 symlink.txt -> file.txt

Отличия от жёсткой ссылки:

ХарактеристикаЖёсткаяСимволическая
Свой inodeНетДа
Другая ФСНетДа
На директориюНетДа
Если оригинал удалёнДанные доступны«Битая» ссылка

VFS (Virtual File System)

VFS — абстрактный слой между ядром и конкретными файловыми системами.

Зачем нужна VFS

  1. Единый интерфейс: все ФС выглядят одинаково
  2. Прозрачность: не важно, ext4, XFS, или NFS
  3. Кэширование: общий кэш для всех ФС

Как работает VFS

Приложение
    ↓
Системный вызов (open, read, write)
    ↓
VFS (виртуальный слой)
    ↓
Драйвер конкретной ФС (ext4, XFS, NFS...)
    ↓
Диск / Сеть

VFS и inode

VFS создаёт виртуальные inode в памяти:

Диск (ext4 inode) → VFS (inode в памяти) → Приложение

Это позволяет:

  • Кэшировать метаданные
  • Работать с разными ФС одновременно
  • Быстро получать информацию о файле

Практическое значение

Удалённый файл занимает место

# Процесс держит файл открытым
cat file.txt &
PID=$!

# Удалить файл
rm file.txt

# Файл «удалён», но место занято
df -h

# Процесс всё ещё читает данные
ls -l /proc/$PID/fd/
# lrwx------ 1 user user 64 Mar 14 10:00 3 -> /home/spiage/file.txt (deleted)

Решение:

# 1. Перезапустить процесс
kill $PID

# 2. Или освободить FD через /proc
truncate -s 0 /proc/$PID/fd/3

Поиск по inode

# Найти файл по номеру inode
find / -inum 1234567

# Полезно когда файл удалён, но процесс держит
lsof | grep deleted

Жёсткие ссылки для резервирования

# Быстрые «копии» без затрат места
ln document.doc backup1.doc
ln document.doc backup2.doc

# Все изменения видны во всех ссылках
echo "update" >> document.doc
cat backup1.doc  # Покажет "update"

Символические ссылки для версионирования

# Активная версия
ln -s app-v2.1.0 app

# При обновлении
ln -sfn app-v2.2.0 app  # -f = заменить существующую

Временные метки

МеткаОписаниеКогда меняется
atimeВремя доступаПри чтении
mtimeВремя модификацииПри изменении содержимого
ctimeВремя изменения inodeПри изменении прав, владельца, содержимого

Проверка:

stat file.txt
# Access: 2026-03-14 10:00:00  # atime
# Modify: 2026-03-14 09:00:00  # mtime
# Change: 2026-03-14 09:00:00  # ctime

Изменение:

# Изменить mtime
touch -m file.txt

# Изменить atime
touch -a file.txt

# Установить конкретное время
touch -t 202601011200 file.txt  # 2026-01-01 12:00
