Специальные файлы

Файлы устройств и другие специальные объекты в Linux.

Типы специальных файлов

ТипПрефиксПримерОписание
Блочное устройствоb/dev/sdaУстройства с буферизацией (диски)
Символьное устройствоc/dev/ttyУстройства без буферизации (терминалы)

Просмотр типа файла

ls -l /dev/sda
# brw-rw---- 1 root disk 8, 0 Mar 14 10:00 /dev/sda
# ↑ 'b' = блочное устройство

ls -l /dev/tty
# crw-rw-rw- 1 root tty 5, 0 Mar 14 10:00 /dev/tty
# ↑ 'c' = символьное устройство

ls -l /var/run/docker.sock
# srw-rw---- 1 root docker 0 Mar 14 10:00 /var/run/docker.sock
# ↑ 's' = сокет

ls -l /tmp/mypipe
# prw-r--r-- 1 user user 0 Mar 14 10:00 /tmp/mypipe
# ↑ 'p' = pipe (FIFO)

Блочные устройства

Устройства с произвольным доступом и буферизацией.

Примеры

УстройствоОписание
/dev/sdaПервый SCSI/SATA диск
/dev/sda1Первый раздел на sda
/dev/nvme0n1NVMe SSD
/dev/mapper/vg-lvLVM логический том
/dev/sr0CD/DVD привод

Использование

# Чтение сырых данных с диска
dd if=/dev/sda of=disk.img bs=4M

# Создание ФС
sudo mkfs.ext4 /dev/sda1

# Монтирование
sudo mount /dev/sda1 /mnt/data

# Проверка здоровья
sudo smartctl -a /dev/sda

Символьные устройства

Устройства с последовательным доступом.

Примеры

УстройствоОписание
/dev/ttyУправляющий терминал процесса
/dev/tty0Консоль
/dev/ttyUSB0USB-serial адаптер
/dev/null«Чёрная дыра» (данные исчезают)
/dev/zeroБесконечный поток нулей
/dev/randomГенератор случайных чисел
/dev/urandomГенератор псевдослучайных чисел
/dev/fullВсегда возвращает ошибку записи

Использование

# Записать нули в файл
dd if=/dev/zero of=file bs=1M count=100

# Прочитать случайные данные
dd if=/dev/urandom of=random.bin bs=1K count=10

# Подавить вывод
command > /dev/null 2>&1

# Заполнить диск (для тестов)
dd if=/dev/zero of=/tmp/fill bs=1M

/dev/null, /dev/zero, /dev/random

/dev/null

«Чёрная дыра» — всё записанное исчезает, чтение возвращает EOF.

# Подавить вывод
command > /dev/null 2>&1

# Очистить файл
> /dev/null
# или
cat /dev/null > file.txt

# Всегда EOF
cat /dev/null
# (ничего)

/dev/zero

Бесконечный поток нулей.

# Создать файл заданного размера
dd if=/dev/zero of=disk.img bs=1M count=1024

# Заполнить память (тест)
cat /dev/zero > /dev/null &
# Быстро! Ядро оптимизирует

# Создать swap файл
dd if=/dev/zero of=/swapfile bs=1M count=4096
mkswap /swapfile
swapon /swapfile

/dev/random и /dev/urandom

Генераторы случайных чисел.

# Сгенерировать случайный пароль
head -c 16 /dev/urandom | base64
# aB3$dE5&fG7*hI9j

# Сгенерировать ключ
dd if=/dev/urandom of=key.bin bs=32 count=1

# Заполнить файл случайными данными
dd if=/dev/urandom of=random.dat bs=1M count=10

Разница:

/dev/random/dev/urandom
БлокируетсяДа, если мало энтропииНет
КачествоВышеДостаточно для большинства задач
РекомендуетсяДля генерации ключейДля всего остального

FIFO (именованные каналы)

Канал для межпроцессного взаимодействия.

# Создать FIFO
mkfifo /tmp/mypipe

# Записать в канал (блокируется пока нет читателя)
echo "hello" > /tmp/mypipe &

# Прочитать из канала
cat /tmp/mypipe
# hello

# Использовать в конвейере
ls -l > /tmp/mypipe &
grep ".txt" < /tmp/mypipe

Особенности:

  • Данные не сохраняются на диск
  • Читатель и писатель должны работать одновременно
  • При закрытии писателя читатель получает EOF

Сокеты

Файлы для сетевого и локального IPC.

Просмотр сокетов

# Unix domain сокеты
ss -xl

# Вывод:
# Netid  State  Recv-Q  Send-Q  Local Address:Port  Peer Address:Port
# u_str  LISTEN 0       128     /var/run/docker.sock 12345  * 0

# TCP сокеты
ss -tl

# UDP сокеты
ss -ul

Подключение к сокету

# Unix сокет
nc -U /var/run/docker.sock

# Отправить запрос
echo "GET /info HTTP/1.0\r\n\r\n" | nc -U /var/run/docker.sock

# TCP сокет
nc localhost 8080

# С таймаутом
nc -w 5 localhost 8080

Создание специальных файлов

# Создать FIFO
mkfifo /tmp/mypipe

# Создать устройство (требует root)
sudo mknod /dev/mydev c 1 3  # символьное, major=1, minor=3
sudo mknod /dev/mydev b 8 1  # блочное, major=8, minor=1

# Удалить
sudo rm /dev/mydev

Major/Minor номера:

  • Major — тип устройства (диск, терминал, и т.д.)
  • Minor — конкретное устройство
# Посмотреть номера
ls -l /dev/sda
# brw-rw---- 1 root disk 8, 0 Mar 14 10:00 /dev/sda
#                        ↑ ↑
#                      major minor

/dev в контейнерах

Контейнеры обычно имеют ограниченный набор устройств:

# В контейнере Docker
docker run --rm alpine ls -l /dev/
# crw-r--r-- 1 root root 5, 1 May 14 00:00 console
# crw-rw-rw- 1 root root 1, 3 May 14 00:00 null
# crw-rw-rw- 1 root root 5, 0 May 14 00:00 tty
# crw-rw-rw- 1 root root 1, 9 May 14 00:00 urandom
# ...

# Пробросить устройство в контейнер
docker run --rm --device=/dev/sda:/dev/sda alpine ls -l /dev/sda

Безопасность

Ограничения

# Запретить доступ к устройствам
chmod 0660 /dev/sda
chown root:disk /dev/sda

# Запретить создание устройств
mount -o nodev /home

Проверка прав

#!/bin/bash

# Найти все устройства с неправильными правами
find /dev -type b -o -type c | while read dev; do
    perms=$(stat -c %a "$dev")
    if [[ "$perms" == "777" ]]; then
        echo "Warning: $dev has permissions $perms"
    fi
done

Контейнеры

# Запустить контейнер без устройств
docker run --rm --device-cgroup-rule='b 8:* rm' alpine

# Только необходимые устройства
docker run --rm --device=/dev/sda alpine
