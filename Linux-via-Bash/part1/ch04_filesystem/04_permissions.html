От основ Linux к мастерству Bash

Права доступа

Механизм контроля доступа к файлам в Linux.

Модель прав доступа

Каждый файл имеет владельца и группу. Права делятся на три категории:

КатегорияОбозначениеКто это
Владелецu (user)Владелец файла
Группаg (group)Члены группы файла
Остальныеo (others)Все остальные

Биты прав

Для каждой категории есть три бита:

БитДля файлаДля директории
r (read)Чтение содержимогоПросмотр списка файлов
w (write)Запись/изменениеСоздание/удаление файлов
x (execute)ВыполнениеВход в директорию

Важно: для удаления файла нужны права на директорию, а не на файл!

Просмотр прав

ls -l /path/to/file

# Пример вывода:
# -rw-r--r-- 1 user group 1234 Mar 14 10:00 file.txt
# ││││││││││
# │││││││││└── Остальные: r-- (читать)
# ││││││││└───── Группа: r-- (читать)
# │││││││└──────── Владелец: rw- (читать+писать)
# ││││││└─────────── Специальные биты
# │││││└───────────── Тип: - (файл), d (директория), l (ссылка)

Типы файлов:

СимволТип
-Обычный файл
dДиректория
lСимволическая ссылка
bБлочное устройство
cСимвольное устройство
pИменованный канал
sСокет

Числовое представление

БитЗначение
r4
w2
x1

Примеры:

  • 755 = rwxr-xr-x (владелец: всё, группа: чтение+выполнение, остальные: чтение+выполнение)
  • 644 = rw-r--r-- (владелец: чтение+запись, остальные: чтение)
  • 600 = rw------- (только владелец читает и пишет)
  • 777 = rwxrwxrwx (все всё могут — опасно!)

Изменение прав

chmod

# Числовое
chmod 755 script.sh
chmod 644 file.txt

# Символическое
chmod u+x script.sh      # Добавить выполнение владельцу
chmod go-w file.txt      # Убрать запись у группы и остальных
chmod a+r file.txt       # Добавить чтение всем
chmod u=rw,go=r file.txt # Точная установка

Рекурсивно:

chmod -R 755 /path/to/dir

chown

# Изменить владельца
chown user file.txt

# Изменить владельца и группу
chown user:group file.txt

# Рекурсивно
chown -R user:group /path/to/dir

chgrp

# Изменить только группу
chgrp group file.txt

Как ядро проверяет права

При открытии файла ядро проверяет:

1. Процесс запущен от root?
   └─→ ДА: доступ разрешён
   └─→ НЕТ: перейти к шагу 2

2. UID процесса = UID владельца?
   └─→ ДА: проверить биты владельца
   └─→ НЕТ: перейти к шагу 3

3. GID процесса в группе файла?
   └─→ ДА: проверить биты группы
   └─→ НЕТ: проверить биты остальных

Пример:

# Файл: -rw-r--r-- (644)
# Владелец: user (UID 1000)
# Группа: user (GID 1000)

# Процесс от user (UID 1000):
# → Проверка битов владельца: rw- → Чтение+запись ✓

# Процесс от other (UID 2000):
# → Проверка битов остальных: r-- → Только чтение ✓

Ошибки доступа

ОшибкаКодКогда
EACCES13Нет прав на файл
EPERM1Операция запрещена (например, chown без root)

Примеры:

# EACCES — файл есть, но нет прав на чтение
chmod 000 /etc/shadow
cat /etc/shadow
# cat: /etc/shadow: Permission denied

strace cat /etc/shadow 2>&1 | grep EACCES
# openat(..., "/etc/shadow", O_RDONLY) = -1 EACCES

# EPERM — операция запрещена
chown root /etc/passwd
# chown: changing ownership of '/etc/passwd': Operation not permitted

umask

umask — маска по умолчанию для новых файлов.

# Посмотреть umask
umask  # Обычно 0022

# Изменить
umask 027  # Новые файлы: 640, директории: 750

Расчёт прав:

  • Файлы: 666 - umask
  • Директории: 777 - umask

Примеры:

umask 022
touch file.txt      # 644 (rw-r--r--)
mkdir newdir        # 755 (rwxr-xr-x)

umask 077
touch secret.txt    # 600 (rw-------)
mkdir private       # 700 (rwx------)

ACL (Access Control Lists)

Расширенная система прав для отдельных пользователей/групп.

# Посмотреть ACL
getfacl file.txt

# Добавить право конкретному пользователю
setfacl -m u:spiage:rx file.txt

# Добавить право группе
setfacl -m g:groupname:rx file.txt

# Удалить ACL
setfacl -b file.txt

Пример вывода:

getfacl /etc/shadow
# file: /etc/shadow
# owner: root
# group: shadow
# user::rw-
# group::---
# other:---
# mask::rw-
# user:admin:r--  # Дополнительное право

Практические рекомендации

1. Минимум привилегий

# ПЛОХО: все всё могут
chmod 777 /var/www

# ХОРОШО: только необходимое
chmod 755 /var/www
chown www-data:www-data /var/www

2. Секреты только для владельца

# Ключи, пароли, токены
chmod 600 ~/.ssh/id_rsa
chmod 600 ~/.netrc
chmod 600 .env

3. Скрипты на выполнение

# Скрипты
chmod 755 script.sh

# Но не данные!
chmod 644 data.txt

4. Sticky для общих папок

# /tmp — удаление только владельцем
chmod +t /tmp

# Или числом
chmod 1777 /tmp

5. setgid для общих директорий

# Файлы наследуют группу
chmod g+s /shared/project
# или
chmod 2775 /shared/project
