От основ Linux к мастерству Bash

Файловые дескрипторы

Файловый дескриптор (FD) — это неотрицательное целое число, которое ядро использует для идентификации открытых файлов процесса.

Что такое FD

Когда процесс открывает файл, ядро возвращает дескриптор — маленькое число:

int fd = open("/etc/hostname", O_RDONLY);
// fd = 3 (например)

Процесс использует это число для всех операций:

read(fd, buffer, size);   // Читать
write(fd, buffer, size);  // Писать
close(fd);                // Закрыть

В bash:

# bash скрывает FD, но они есть
echo "hello" > file.txt
# 1. open("file.txt", O_WRONLY|O_CREAT) = 3
# 2. write(3, "hello\n", 7) = 7
# 3. close(3) = 0

Стандартные дескрипторы

Каждый процесс наследует три стандартных FD:

FDПотокУстройство по умолчанию
0stdinКлавиатура (терминал)
1stdoutТерминал (экран)
2stderrТерминал (экран)

Пример в bash

# stdin (0) — ввод
cat < input.txt

# stdout (1) — вывод
cat file.txt > output.txt

# stderr (2) — ошибки
command 2> errors.log

# Все три вместе
command < in.txt > out.txt 2> err.log

Проверка FD

# Посмотреть FD процесса
ls -l /proc/$$/fd/

# Вывод:
# lrwx------ 1 user user 64 Mar 14 10:00 0 -> /dev/pts/0
# lrwx------ 1 user user 64 Mar 14 10:00 1 -> /dev/pts/0
# lrwx------ 1 user user 64 Mar 14 10:00 2 -> /dev/pts/0

Таблица файловых дескрипторов

У каждого процесса есть таблица FD — массив, где индекс = номер дескриптора, а значение = указатель на запись в системной таблице открытых файлов.

Процесс A:
  FD 0 → /dev/pts/0 (stdin)
  FD 1 → /dev/pts/0 (stdout)
  FD 2 → /dev/pts/0 (stderr)
  FD 3 → /etc/hostname
  FD 4 → /var/log/app.log

Проверка:

# Посмотреть таблицу FD
ls -l /proc/<pid>/fd/

# Пример вывода:
# lrwx------ 1 user user 64 Mar 14 10:00 0 -> /dev/pts/0
# lrwx------ 1 user user 64 Mar 14 10:00 1 -> /dev/pts/0
# lrwx------ 1 user user 64 Mar 14 10:00 2 -> /dev/pts/0
# lrwx------ 1 user user 64 Mar 14 10:00 3 -> /etc/hostname
# lrwx------ 1 user user 64 Mar 14 10:00 4 -> /var/log/app.log
# lrwx------ 1 user user 64 Mar 14 10:00 5 -> socket:[12345]

Наследование FD

При создании процесса (fork) дочерний процесс наследует все FD родителя:

#!/bin/bash

# Родитель открывает файл
exec 3> /tmp/test.log

# Дочерний процесс наследует FD 3
bash -c 'echo "from child" >&3'

# FD 3 доступен в обоих процессах
echo "from parent" >&3

# Закрыть FD 3
exec 3>&-

Проверка наследования:

# Родитель
bash -c 'ls -l /proc/$$/fd/' &
PID=$!

# Дочерний процесс
bash -c "ls -l /proc/$PID/fd/"

# FD одинаковые!

Дублирование FD

Можно создать копию дескриптора:

В bash

# Сохранить stdout в FD 3
exec 3>&1

# Перенаправить stdout в файл
exec > output.txt

# Писать в файл
echo "to file"

# Восстановить stdout из FD 3
exec 1>&3

# Писать в терминал
echo "to terminal"

# Закрыть FD 3
exec 3>&-

Через /proc

# Открыть копию FD другого процесса
cat /proc/<pid>/fd/3

# Или скопировать
cp /proc/<pid>/fd/3 ./copied_fd

Лимиты FD

У каждого процесса есть лимит на количество открытых FD:

# Посмотреть лимиты
ulimit -n  # Мягкий лимит (например, 1024)
ulimit -Hn # Жёсткий лимит (например, 4096)

# Изменить (в сессии)
ulimit -n 4096

Системный лимит:

# Максимум FD для всех процессов
cat /proc/sys/fs/file-max
# 100000

# Текущее использование
cat /proc/sys/fs/file-nr
# 10000  0  100000
# ↑      ↑  ↑
# открыто unused максимум

Ошибки с FD

ОшибкаКодПричина
EBADF9Некорректный FD (закрыт или не существовал)
EMFILE24Достигнут лимит FD процесса
ENFILE23Достигнут системный лимит открытых файлов

Примеры:

# EBADF — закрытый FD
exec 3>&-
echo "test" >&3
# bash: 3: Bad file descriptor

# EMFILE — слишком много FD
for i in {1..10000}; do
    exec {fd}>&1  # Открыть новый FD
done
# bash: ulimit: open files: cannot allocate memory

Практические примеры

Перенаправление вывода в скрипте

#!/bin/bash

# Логирование
exec > >(tee -a script.log) 2>&1

echo "Starting..."
process_data
echo "Completed"

Сохранение и восстановление stdout

#!/bin/bash

# Сохранить оригинальный stdout
exec 3>&1

# Перенаправить в файл
exec > output.txt

# ... команды пишут в файл ...
echo "to file"

# Восстановить stdout
exec 1>&3
exec 3>&-

# ... команды пишут в терминал ...
echo "to terminal"

Проверка открытых FD

#!/bin/bash

check_fd_leak() {
    local pid=$1
    local prev=$(ls /proc/$pid/fd/ | wc -l)
    
    sleep 10
    
    local curr=$(ls /proc/$pid/fd/ | wc -l)
    
    if [[ $curr -gt $prev ]]; then
        echo "Possible FD leak: $prev -> $curr"
        ls -l /proc/$pid/fd/
    fi
}

Чтение из конкретного FD

#!/bin/bash

# Открыть файл на FD 3
exec 3< input.txt

# Читать с FD 3
while read -r line <&3; do
    echo "Read: $line"
done

# Закрыть FD 3
exec 3<&-
