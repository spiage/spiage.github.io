От основ Linux к мастерству Bash

Основы процессов

Процесс — это базовая единица выполнения в Linux.

Что такое процесс

Процесс — экземпляр выполняющейся программы. Содержит:

  • Код программы (инструкции)
  • Данные (переменные, буферы)
  • Ресурсы (открытые файлы, сетевые подключения)
  • Состояние (значения регистров, стек)

PCB (Process Control Block)

Ядро хранит информацию о каждом процессе в структуре PCB:

ПолеОписание
PIDИдентификатор процесса
PPIDИдентификатор родителя
UID/GIDВладелец и группа
Состояниеrunning, sleeping, и т.д.
Таблица FDОткрытые файловые дескрипторы
ПамятьУказатели на код, данные, стек
РегистрыЗначения CPU регистров
ПриоритетPriority, nice value

PID и PPID

# Посмотреть PID текущего процесса
echo $$

# Посмотреть PPID (родительский процесс)
echo $PPID

# Посмотреть все процессы
ps aux

# Дерево процессов
pstree

Иерархия:

init/systemd (PID 1)
├── sshd (PID 100)
│   └── bash (PID 1000)
│       └── vim (PID 1001)
└── cron (PID 200)

Атрибуты процесса

UID и GID

АтрибутОписание
Real UIDРеальный владелец (кто запустил)
Effective UIDЭффективный владелец (для проверок прав)
Saved UIDСохранённый UID (для setuid программ)
 
# Посмотреть UID/GID процесса
ps -o pid,user,uid,gid,comm

# Пример вывода:
#   PID USER       UID   GID COMMAND
#  1234 user      1000  1000 bash

Группы процессов

Процессы объединяются в группы для управления сигналами:

# Посмотреть PGID (Process Group ID)
ps -o pid,pgid,comm

# Отправить сигнал всей группе
kill -- -PGID  # Отрицательный PID = группа

Просмотр информации о процессе

# Основная информация
ps -p <pid> -o pid,ppid,user,state,comm

# Детальная информация через /proc
cat /proc/<pid>/status

# Аргументы командной строки
cat /proc/<pid>/cmdline | tr '\0' ' '

# Окружение процесса
cat /proc/<pid>/environ | tr '\0' '\n'

# Открытые файлы
ls -l /proc/<pid>/fd/

# Использование памяти
cat /proc/<pid>/status | grep Vm

Иерархия процессов

Родительские и дочерние процессы

# Запустить дочерний процесс
bash -c 'echo "Child PID: $$, Parent PID: $PPID"'

# Вывод:
# Child PID: 5678, Parent PID: 1234

Сиротские процессы

Когда родитель завершается раньше ребёнка:

# Родитель завершается
bash -c 'sleep 10 &'

# Дочерний процесс становится сиротой
# Его принимает init/systemd (PID 1)

Зомби-процессы

Процесс завершён, но родитель не прочитал статус:

# Статус Z (zombie)
ps aux | grep Z

# Пример:
# USER   PID  S  CMD
# user  1234  Z  [defunct]

Решение: родитель должен вызвать wait().

Влияние на скрипты

Фоновые процессы

# Запустить в фоне
long_task &

# PID фонового процесса
echo $!

# Дождаться завершения
wait $!

Проверка существования процесса

# Проверить существует ли процесс
if kill -0 <pid> 2>/dev/null; then
    echo "Process exists"
fi
