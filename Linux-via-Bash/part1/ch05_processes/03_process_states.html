Жизненный цикл и состояния процессов

Процессы проходят через различные состояния в течение жизни.

Состояния процесса

СостояниеКодОписание
RunningRВыполняется или готов к выполнению
Interruptible SleepSЖдёт событие (можно прервать)
Uninterruptible SleepDЖдёт I/O (нельзя прервать)
StoppedTОстановлен сигналом
ZombieZЗавершён, не убран родителем
DeadXМёртв (не отображается в ps)

Running (R)

Процесс выполняется или в очереди на выполнение:

# Найти running процессы
ps aux | grep ' R '

# Пример:
# USER   PID  S  CMD
# user  1234  R  python3 script.py

Interruptible Sleep (S)

Процесс ждёт событие и может быть прерван сигналом:

# Большинство спящих процессов в S
ps aux | grep ' S ' | wc -l

# Пример:
# USER   PID  S  CMD
# user  5678  S  bash
# user  5679  S  ssh user@host

Почему прерываемый:

  • Ждёт ввод с терминала
  • Ждёт данные из сети
  • Ждёт завершение дочернего процесса

Uninterruptible Sleep (D)

Процесс ждёт I/O и не может быть прерван:

# Найти процессы в D
ps aux | grep ' D '

# Пример:
# USER   PID  S  CMD
# root   123  D  [kworker/u4:0]

Причины D-состояния:

  • Ожидание ответа от диска
  • Ожидание сетевого ресурса
  • Блокировка ядра

Проблема: kill -9 не работает!

# kill игнорируется
kill -9 <pid>  # Ничего не происходит

# Решение: исправить I/O проблему или перезагрузиться

Stopped (T)

Процесс остановлен сигналом:

# Остановить процесс
kill -STOP <pid>

# Проверить
ps aux | grep ' T '

# Продолжить выполнение
kill -CONT <pid>

Причины остановки:

  • Сигнал SIGSTOP, SIGTSTP
  • Отладка (ptrace)
  • Job control (Ctrl+Z)

Zombie (Z)

Процесс завершён, но родитель не прочитал статус:

# Найти зомби
ps aux | grep ' Z '

# Пример:
# USER   PID  S  CMD
# user  9999  Z  [defunct]

Почему возникают:

  1. Дочерний процесс завершается
  2. Родитель не вызывает wait()
  3. Ядро хранит статус в таблице процессов

Кто убирает зомби:

  • Родитель должен вызвать wait()
  • Если родитель умер — zombie принимает init (PID 1)

Решение:

# Найти родителя зомби
ps -o pid,ppid,state,comm | grep -B1 Z

# Отправить родителю SIGCHLD
kill -CHLD <ppid>

# Или убить родителя
kill <ppid>

Влияние на скрипты

Избежание зомби

#!/bin/bash

# Ловушка на SIGCHLD
trap 'wait -n' SIGCHLD

# Запустить детей
for i in {1..10}; do
    (sleep $RANDOM) &
done

# Ждать всех
wait

Обработка D-состояния

#!/bin/bash

# Запустить с таймаутом
timeout 30 dd if=/dev/sda of=/dev/null &
PID=$!

# Проверять состояние
while kill -0 $PID 2>/dev/null; do
    STATE=$(ps -o state= -p $PID)
    if [[ $STATE == "D" ]]; then
        echo "Warning: Process in D state"
    fi
    sleep 5
done

Мониторинг состояний

#!/bin/bash

# Статистика состояний
ps aux | awk '{print $8}' | sort | uniq -c | sort -rn

# Пример вывода:
#   150 S
#    10 R
#     2 D
#     1 Z

Переходы между состояниями

                    ┌─────────┐
              ┌─────│ Running │─────┐
              │     └────┬────┘     │
              │          │          │
         sleep/read      │     exit/kill
              │          │          │
              ▼          ▼          ▼
        ┌─────────┐ ┌──────┐   ┌────────┐
        │   S     │ │  T   │   │ Zombie │
        └────┬────┘ └──┬───┘   └───┬────┘
             │         │           │
        signal/     CONT       wait()
        I/O ready     │           │
             │        │           ▼
             └────────┴─────→ Dead

Практические рекомендации

  1. Всегда используйте wait для фоновых процессов
  2. Избегайте kill -9 — используйте SIGTERM сначала
  3. Мониторьте D-состояния — признак проблем с I/O
  4. Обрабатывайте SIGCHLD в долгих скриптах
