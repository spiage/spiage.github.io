От основ Linux к мастерству Bash

Защита памяти

Механизмы защиты от эксплуатации уязвимостей.

DEP/NX (Data Execution Prevention)

Что делает

DEP/NX запрещает выполнение кода из страниц памяти с данными.

Память:
┌─────────────────┐
│ Код (executable)│ ← Можно выполнять
├─────────────────┤
│ Данные          │ ← Нельзя выполнять
│ (stack, heap)   │
└─────────────────┘

Проверка

# Проверить поддержку NX
grep -i nx /proc/cpuinfo

# Проверить статус для процесса
cat /proc/$$/status | grep Nmib

Влияние на скрипты

# Невозможно выполнить код из стека
# Защита от shellcode в переполнении буфера

ASLR (Address Space Layout Randomization)

Что делает

ASLR рандомизирует расположение памяти:

  • Адреса библиотек
  • Адрес стека
  • Адрес кучи
Без ASLR:          С ASLR:
Код:    0x08048000  Код:    0x08048000
Libc:   0xb7e00000  Libc:   0xb7a00000 (рандом)
Stack:  0xbffff000  Stack:  0xbf900000 (рандом)

Проверка

# Статус ASLR
cat /proc/sys/kernel/randomize_va_space

# 0 = отключено
# 1 = частичное (stack, mmap)
# 2 = полное (включая heap)

Включение

# Включить ASLR
echo 2 > /proc/sys/kernel/randomize_va_space

# Сохранить
echo "kernel.randomize_va_space = 2" >> /etc/sysctl.conf

Отключение (для отладки)

# Отключить для отладки
echo 0 > /proc/sys/kernel/randomize_va_space

# Или для конкретного процесса
setarch $(uname -m) -R ./program

Stack Canaries

Что делает

Stack canary — случайное значение перед адресом возврата в стеке.

Стек:
┌─────────────────┐
│ Локальные       │
│ переменные      │
├─────────────────┤
│ Canary          │ ← Проверяется перед return
├─────────────────┤
│ Адрес возврата  │
└─────────────────┘

При переполнении:

  1. Canary затирается
  2. Перед return проверка
  3. Если changed → abort()

Проверка компиляции

# GCC с защитой
gcc -fstack-protector-all program.c

# Без защиты
gcc -fno-stack-protector program.c

Влияние на скрипты

# Бинарники bash с canaries
readelf -s /bin/bash | grep stack_chk

# Если есть __stack_chk_fail — защита включена

CFI (Control Flow Integrity)

Что делает

CFI проверяет что поток выполнения соответствует ожидаемому графу вызовов.

Вызов функции:
1. Проверить что адрес в таблице вызовов
2. Проверить тип аргументов
3. Разрешить вызов

Проверка

# Проверить CFI в бинарнике
readelf -s /bin/bash | grep cfi

# Или через objdump
objdump -d /bin/bash | grep -i "cfi"

PIE (Position Independent Executable)

Что делает

PIE позволяет загружать исполняемый файл по случайному адресу.

Без PIE:          С PIE:
Код: 0x08048000   Код: 0x55a00000 (рандом)

Проверка

# Проверить PIE
file /bin/bash
# ELF 64-bit LSB pie executable

# Или через readelf
readelf -h /bin/bash | grep Type
# Type: DYN (Shared object file) = PIE включен

Практические примеры

Проверка защит бинарника

#!/bin/bash

check_protections() {
    local binary="$1"
    
    echo "=== $binary ==="
    
    # PIE
    if readelf -h "$binary" 2>/dev/null | grep -q "DYN"; then
        echo "PIE: ✓"
    else
        echo "PIE: ✗"
    fi
    
    # Stack canary
    if readelf -s "$binary" 2>/dev/null | grep -q "__stack_chk"; then
        echo "Stack Canary: ✓"
    else
        echo "Stack Canary: ✗"
    fi
    
    # RELRO
    if readelf -l "$binary" 2>/dev/null | grep -q "GNU_RELRO"; then
        echo "RELRO: ✓"
    else
        echo "RELRO: ✗"
    fi
    
    # NX
    if readelf -l "$binary" 2>/dev/null | grep -q "GNU_STACK.*RW"; then
        echo "NX: ✗ (executable stack)"
    else
        echo "NX: ✓"
    fi
}

check_protections /bin/bash

Аудит системы

#!/bin/bash

echo "=== System Protections ==="

# ASLR
echo -n "ASLR: "
case $(cat /proc/sys/kernel/randomize_va_space) in
    0) echo "Disabled" ;;
    1) echo "Partial" ;;
    2) echo "Full" ;;
esac

# DEP
echo -n "DEP/NX: "
if grep -q nx /proc/cpuinfo; then
    echo "Supported"
else
    echo "Not supported"
fi

# Проверка критичных бинарников
for bin in /bin/bash /usr/bin/sudo /usr/bin/passwd; do
    if [[ -x "$bin" ]]; then
        echo ""
        check_protections "$bin"
    fi
done

Компиляция с защитами

#!/bin/bash

# Компиляция со всеми защитами
gcc \
    -fstack-protector-all \
    -D_FORTIFY_SOURCE=2 \
    -fPIE -pie \
    -Wl,-z,relro,-z,now \
    -o program program.c

# Проверка
checksec --file=program

Ограничения

Что не защищают

  1. Логические уязвимости — неправильная бизнес-логика
  2. Инъекции — SQL, command injection
  3. TOCTOU — Time-of-check-time-of-use
  4. Социальная инженерия — фишинг и т.д.
