    Homelab: контейнеры, ВМ, сеть¶

    Homelab -- это домашняя лаборатория: набор сервисов, контейнеров, виртуальных машин и сетевых хранилищ, работающих на одном или нескольких компьютерах. Бэкап homelab сложнее десктопного -- здесь несколько слоёв, каждый со своими требованиями к консистентности.

    Архитектура типичного homelab¶

    ┌─────────────────────────────────────────┐
│              Homelab Host               │
│  ┌──────────┐  ┌──────────┐  ┌───────┐ │
│  │ Docker   │  │ KVM/QEMU │  │ NFS   │ │
│  │ Podman   │  │ VirtualBox│  │ SMB   │ │
│  └────┬─────┘  └────┬─────┘  └───┬───┘ │
│       │              │            │     │
│  ┌────┴─────┐  ┌────┴─────┐  ┌───┴───┐ │
│  │ Volumes  │  │ .qcow2   │  │ Shares│ │
│  │ Networks │  │ XML conf │  │ Exports│ │
│  └──────────┘  └──────────┘  └───────┘ │
│                                         │
│  ┌──────────────────────────────────┐  │
│  │       Базы данных (БД)          │  │
│  │  PostgreSQL  │  MySQL  │ Redis  │  │
│  └──────────────────────────────────┘  │
└─────────────────────────────────────────┘

    Для полного восстановления homelab нужно бэкапить все четыре слоя: 1. Docker/Podman -- compose-файлы, volumes, сети 2. Виртуальные машины -- disk images, конфигурации 3. Сетевые сервисы -- NFS exports, SMB shares 4. Базы данных -- дампы БД (не файлы!)

    Docker / Podman: бэкап контейнеров¶

    Что бэкапить¶

    Компонент Где находится Зачем
    Docker Compose файлы ~/docker-compose.yml, ~/projects/*/docker-compose.yml Описание сервисов, версий, портов
    Docker volumes /var/lib/docker/volumes/ Данные контейнеров (БД, файлы)
    Docker networks Определяются в compose Сетевая конфигурация
    .env файлы Рядом с compose Секреты, переменные окружения

    Бэкап Docker volume¶

    Самый безопасный способ -- через временный контейнер:

    # Бэкап Docker volume
$ docker run --rm \
    -v my_volume_name:/data:ro \
    -v $(pwd):/backup \
    alpine \
    tar czf /backup/volume-backup-$(date +%F).tar.gz -C /data .

    Разберём команду:

    Флаг Что делает
    --rm Удаляет контейнер после завершения
    -v my_volume_name:/data:ro Монтирует volume в /data (read-only)
    -v $(pwd):/backup Монтирует текущую директорию в /backup
    alpine Лёгкий образ для запуска tar
    tar czf /backup/... Архивирует данные volume

    Восстановление Docker volume¶

    # Создаём volume (если не существует)
$ docker volume create my_volume_name

# Восстанавливаем из бэкапа
$ docker run --rm \
    -v my_volume_name:/data \
    -v $(pwd):/backup \
    alpine \
    tar xzf /backup/volume-backup-2026-04-03.tar.gz -C /data

    Бэкап всех volumes¶

    #!/bin/bash
# backup-all-docker-volumes.sh

BACKUP_DIR="/backup/docker-volumes"
mkdir -p "$BACKUP_DIR"

# Получаем список всех volumes
for vol in $(docker volume ls -q); do
    echo "Бэкап volume: $vol"
    docker run --rm \
        -v "${vol}:/data:ro" \
        -v "${BACKUP_DIR}:/backup" \
        alpine \
        tar czf "/backup/${vol}-$(date +%F).tar.gz" -C /data .
done

echo "=== Готово ==="
ls -lh "$BACKUP_DIR"

    Бэкап Docker Compose стека¶

    #!/bin/bash
# backup-compose-stack.sh
# Бэкап compose-файлов, .env и volumes

PROJECT_DIR="/opt/myapp"
BACKUP_DIR="/backup/compose-$(date +%F)"

mkdir -p "$BACKUP_DIR"

# Копируем compose-файлы и конфиги
cp -r "$PROJECT_DIR" "$BACKUP_DIR/project"

# Бэкапим volumes
for vol in $(docker volume ls -f "name=${PROJECT_DIR##*/}" -q); do
    docker run --rm \
        -v "${vol}:/data:ro" \
        -v "${BACKUP_DIR}:/backup" \
        alpine \
        tar czf "/backup/${vol}.tar.gz" -C /data .
done

# Сохраняем список запущенных контейнеров
docker compose -f "$PROJECT_DIR/docker-compose.yml" ps > "$BACKUP_DIR/containers-status.txt"

echo "Бэкап стека: $BACKUP_DIR"

    Базы данных: дампы, а не файлы¶

    Критично: Базы данных нельзя бэкапить простым копированием файлов данных. Файлы БД находятся в постоянном изменении, и простая копия может дать битые, неконсистентные данные. Используйте утилиты дампа.

    PostgreSQL¶

    # Дамп одной базы
$ docker exec postgres_container pg_dump -U postgres mydb \
    > /backup/postgres-mydb-$(date +%F).sql

# Дамп всех баз
$ docker exec postgres_container pg_dumpall -U postgres \
    | gzip > /backup/postgres-all-$(date +%F).sql.gz

# Восстановление
$ docker exec -i postgres_container psql -U postgres mydb \
    < /backup/postgres-mydb-2026-04-03.sql

    MySQL / MariaDB¶

    # Дамп базы
$ docker exec mysql_container mysqldump -u root -pPASSWORD mydb \
    | gzip > /backup/mysql-mydb-$(date +%F).sql.gz

# Дамп всех баз
$ docker exec mysql_container mysqldump -u root -pPASSWORD --all-databases \
    | gzip > /backup/mysql-all-$(date +%F).sql.gz

# Восстановление
$ gunzip < /backup/mysql-mydb-2026-04-03.sql.gz \
    | docker exec -i mysql_container mysql -u root -pPASSWORD mydb

    SQLite¶

    # Дамп SQLite (для контейнеров с SQLite)
$ docker exec app_container sqlite3 /data/app.db ".dump" \
    > /backup/sqlite-app-$(date +%F).sql

# Или просто копируем файл (SQLite -- один файл, можно копировать при остановленном контейнере)
$ docker stop app_container
$ docker cp app_container:/data/app.db /backup/sqlite-app-$(date +%F).db
$ docker start app_container

    KVM / QEMU: бэкап виртуальных машин¶

    Что бэкапить¶

    Для каждой ВМ нужно сохранить: 1. Конфигурацию (XML-описание) 2. Дисковые образы (.qcow2 или .raw)

    Экспорт конфигурации ВМ¶

    # Экспорт XML-конфигурации
$ virsh dumpxml my-vm > /backup/vm-my-vm-$(date +%F).xml

# Экспорт всех ВМ
$ for vm in $(virsh list --all --name); do
    virsh dumpxml "$vm" > "/backup/vm-${vm}-$(date +%F).xml"
done

    Бэкап дисковых образов¶

    # Находим disk images ВМ
$ virsh domblklist my-vm
Target   Source
-----------------------------
vda      /var/lib/libvirt/images/my-vm.qcow2
vdb      /var/lib/libvirt/images/my-vm-data.qcow2

# Бэкап с snapshot (без остановки ВМ)
$ virsh snapshot-create-as my-vm backup-snapshot --disk-only --quiesce

# Копируем disk image
$ cp /var/lib/libvirt/images/my-vm.qcow2 \
    /backup/vm-my-vm-disk-$(date +%F).qcow2

# Удаляем снапшот
$ virsh snapshot-delete my-vm backup-snapshot

    Бэкап с qemu-img (консистентный)¶

    # Создание бэкапа через qemu-img (требует поддержки)
$ qemu-img create -f qcow2 -b /var/lib/libvirt/images/my-vm.qcow2 \
    /backup/vm-my-vm-backup.qcow2

# Или конвертация в сжатый образ
$ qemu-img convert -c -O qcow2 \
    /var/lib/libvirt/images/my-vm.qcow2 \
    /backup/vm-my-vm-$(date +%F).qcow2

    Восстановление ВМ¶

    # Восстановление конфигурации
$ virsh define /backup/vm-my-vm-2026-04-03.xml

# Восстановление disk image
$ cp /backup/vm-my-vm-disk-2026-04-03.qcow2 \
    /var/lib/libvirt/images/my-vm.qcow2

# Запуск
$ virsh start my-vm

    VirtualBox: бэкап виртуальных машин¶

    # Экспорт ВМ в OVA (универсальный формат)
$ VBoxManage export my-vm -o /backup/vm-my-vm-$(date +%F).ova

# Импорт
$ VBoxManage import /backup/vm-my-vm-2026-04-03.ova

# Или клонирование disk image
$ VBoxManage clonehd /path/to/my-vm.vdi /backup/my-vm-clone.vdi --format VDI

    NFS / SMB: бэкап сетевых хранилищ¶

    NFS¶

    # Бэкап конфигурации NFS
$ sudo tar czf /backup/nfs-config-$(date +%F).tar.gz \
    /etc/exports \
    /etc/exports.d/ \
    /etc/idmapd.conf

# Просмотр экспортированных шар
$ showmount -e localhost

# Бэкап данных NFS-шары
$ sudo rsync -aAXv /srv/nfs/ /backup/nfs-data-$(date +%F)/

    SMB (Samba)¶

    # Бэкап конфигурации Samba
$ sudo tar czf /backup/samba-config-$(date +%F).tar.gz \
    /etc/samba/smb.conf \
    /etc/samba/smb.conf.d/

# Бэкап данных SMB-шар
$ sudo rsync -aAXv /srv/samba/ /backup/samba-data-$(date +%F)/

# Бэкап пользователей Samba (если есть)
$ sudo pdbedit -L > /backup/samba-users-$(date +%F).txt

    Полный скрипт бэкапа homelab¶

    #!/bin/bash
# backup-homelab.sh -- полный бэкап homelab
set -euo pipefail

BACKUP_DIR="/backup/homelab-$(date +%F)"
mkdir -p "$BACKUP_DIR"/{docker,kvm,nfs,samba,databases}

echo "=== Docker volumes ==="
for vol in $(docker volume ls -q); do
    docker run --rm \
        -v "${vol}:/data:ro" \
        -v "${BACKUP_DIR}/docker:/backup" \
        alpine tar czf "/backup/${vol}.tar.gz" -C /data .
done

# Compose-файлы
if [ -d ~/docker ]; then
    cp -r ~/docker "$BACKUP_DIR/docker/compose-files"
fi

echo "=== Базы данных ==="
# PostgreSQL
if docker ps --format '{{.Names}}' | grep -q postgres; then
    docker exec postgres pg_dumpall -U postgres \
        | gzip > "$BACKUP_DIR/databases/postgres-all.sql.gz"
fi

# MySQL
if docker ps --format '{{.Names}}' | grep -q mysql; then
    docker exec mysql mysqldump -u root -p"${MYSQL_ROOT_PASSWORD}" --all-databases \
        | gzip > "$BACKUP_DIR/databases/mysql-all.sql.gz"
fi

echo "=== KVM ВМ ==="
for vm in $(virsh list --all --name); do
    virsh dumpxml "$vm" > "$BACKUP_DIR/kvm/vm-${vm}.xml"
    for disk in $(virsh domblklist "$vm" --details | grep file | awk '{print $4}'); do
        disk_name=$(basename "$disk")
        cp "$disk" "$BACKUP_DIR/kvm/${disk_name}"
    done
done

echo "=== NFS/SMB конфиги ==="
sudo tar czf "$BACKUP_DIR/nfs/nfs-config.tar.gz" /etc/exports /etc/exports.d/ 2>/dev/null || true
sudo tar czf "$BACKUP_DIR/samba/samba-config.tar.gz" /etc/samba/ 2>/dev/null || true

echo "=== Контрольные суммы ==="
find "$BACKUP_DIR" -type f -name "*.tar.gz" -o -name "*.sql.gz" -o -name "*.qcow2" \
    -exec sha256sum {} \; > "$BACKUP_DIR/checksums.sha256"

echo "=== Готово ==="
du -sh "$BACKUP_DIR"

    Проверка работоспособности сервисов после восстановления¶

    После восстановления homelab нужно убедиться, что все сервисы работают:

    # Docker контейнеры запущены?
$ docker ps --format "table {{.Names}}\t{{.Status}}\t{{.Ports}}"

# HTTP-сервисы отвечают?
$ curl -s -o /dev/null -w "%{http_code}" http://localhost:8080/health
200

# База данных доступна?
$ pg_isready -h localhost -p 5432
localhost:5432 - accepting connections

# MySQL
$ mysqladmin -u root -pPASSWORD ping
mysqld is alive

# NFS экспорты
$ showmount -e localhost

# SMB шары
$ smbclient -L localhost -U%

# KVM ВМ запущены?
$ virsh list --state-running

    Сводная таблица: что бэкапить в homelab¶

    Компонент Что бэкапить Чем Частота
    Docker compose файлы .yml, .env cp, rsync При изменении
    Docker volumes Данные volumes docker run + tar Ежедневно
    PostgreSQL Дамп БД pg_dump, pg_dumpall Ежедневно/ежечасно
    MySQL Дамп БД mysqldump Ежедневно/ежечасно
    KVM ВМ XML + .qcow2 virsh dumpxml, qemu-img Еженедельно
    NFS /etc/exports + данные tar, rsync При изменении
    SMB smb.conf + данные tar, rsync При изменении

    Типичные ошибки¶

    Ошибка Последствие Как избежать
    Копирование файлов БД вместо дампа Битые, неконсистентные данные Используйте pg_dump, mysqldump
    Забытые compose-файлы Невозможно воссоздать стек Бэкапьте compose-файлы отдельно
    Бэкап volume без остановки контейнера Неполные данные Используйте read-only монтирование или остановите контейнер
    Забытые .env файлы Нет секретов и переменных Включайте .env в бэкап
    Отсутствие проверки после восстановления Сервисы не работают, но вы не знаете Проверяйте через curl, pg_isready, docker ps
